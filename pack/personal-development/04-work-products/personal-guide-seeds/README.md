# Заготовки персонального руководства

> Набор типовых фрагментов, из которых скилл `/personal-guide-start` собирает первую версию репо `personal-guide/{user}`. Заготовки — не финальный документ; они подставляются под RCS-профиль ученика и доменный контекст, а затем достраиваются в диалоге Claude Code.

## Руководство (guide) vs Табло (panel) — контракт назначения

> **Блокирующее различение (директива пилота 2026-06-30).** Персональное руководство — пара файлов с РАЗНЫМ назначением. Путать их — корневая ошибка генератора.

| Файл | Что это | Что внутри |
|------|---------|------------|
| **`guide/<дата>.md` — руководство** | Сам **учебный текст для изучения**: содержательный материал по теме дня, который ученик читает и осваивает (как адаптированная глава). | Знание, объяснение, разбор по теме. Насыщенный текст. |
| **`panel/<дата>.md` — табло** | **Организационно-статистический срез и задания**: что делать, показатели, состояние. | Ступень, узкое место, RCS-показатели, список РП, активность, задания/«сделай сегодня». |

**Тест маршрутизации содержимого:** «Это текст, который человек ИЗУЧАЕТ, или указание, что СДЕЛАТЬ / цифра о его состоянии?» Изучает → `guide/`. Делает/цифра/оргинфо → `panel/`.

**Запрещено в `guide/`:** задания («задание на моделирование», «шаг 1/2/3», «выпусти v0.1», «зафиксируй в history»), показатели, статистика активности, состав программ, список РП отдельным разделом. Всё это — в `panel/`.

**Объём:** учебный текст руководства должен быть насыщенным; сокращать его ради «чистоты» — ошибка (чистят табло от прозы, не руководство от содержания).

**Зависимость от источника:** наполнить руководство настоящим учебным текстом можно только при подключённом материале программ (ЛР/РР/ИР). Без источника генератор не должен выдумывать содержание — это сигнал подключить источник.

## Топология: хранение / сборка / отображение (ADR-2026-06-30)

> **Архитектурное решение (ArchGate PASS).** Полный текст: `DS-ecosystem-development/.../C2.2.Architecture/ADR-2026-06-30-personal-guide-topology.md`. Здесь — доменная выжимка.

Разделяем **метод** ≠ **место исполнения / резидентность данных** ≠ **поверхность**:

| Слой | Где | Правило |
|------|-----|---------|
| **Хранение общего** (универсальные руководства, seed, знания) | Платформа | Канон = Pack; скомпилированное = единое хранилище платформы (read-only). Все читают одно; не дублируется по пользователям. |
| **Метод сборки** (логика Портного) | Платформа (единый), раздаётся клиентам | Один метод, тянется обновлением (не копия руками → нет дрейфа). |
| **Исполнение + личные данные** | По зоне доверия | Нет IWE → сборка на платформе. Есть IWE → сборка локально, личные данные (профиль/RCS/контекст/рефлексии) не уходят на платформу; тянется только общий материал + метод. |
| **Отображение** | Витрина + опц. репо | Витрина `guide.system-school.ru` у всех; IWE-пользователь подключает свой репо, публикуется только `guide/` (готовое руководство), не сырые данные. |

**Ключ:** единый — метод, а не место. Два пути исполнения (платформа/локально) — из приватности, не дублирование логики.

## Назначение

- **Вход:** RCS-профиль ученика (PD.FORM.089) + bottleneck + M3-домен.
- **Выход:** три файла в репо пользователя — `profile.md`, `worldview.md`, `methods.md` (core-слой), плюс первые `weekly/<year-week>.md` и `daily/<date>.md`.
- **Границы:** заготовки фиксируют ступень × домен на момент bootstrap. Дальнейшая адаптация — через обычный чат Claude Code (пилот просит обновления, Claude Code читает core и history, переписывает weekly/daily).

## Структура

| Ось | Файлы | Откуда берутся данные |
|-----|-------|-----------------------|
| **Ступень** | [stage-2-practicing.md](stage-2-practicing.md), [stage-3-systematic.md](stage-3-systematic.md), [stage-4-disciplined.md](stage-4-disciplined.md) | PD.FORM.003 (определения ступеней) + PD.FORM.089 (RCS-stage = min(W, M1, M2, M4).baseline) |

**Правило композиции:** одна ступенная заготовка + одна доменная вставка = core-слой репо. Ступень определяет, какие методы и мировоззренческие сдвиги подавать; домен определяет язык и примеры.

## Ступени вне MVP

- **Случайный (ступень 1):** у такого ученика нет устойчивого ритма — core-руководство преждевременно. Bootstrap вместо ступенной заготовки отдаёт короткий `profile.md` с одним вопросом «Какая неудовлетворённость привела тебя сюда?» и откладывает остальное до первой сессии.
- **Проактивный (ступень 5):** выходит за рамки программы ЛР. Бут-скилл отправляет к программе «Рабочее развитие» (WP-194). Заготовка не требуется.

## Запреты Pack (Hard Bans)

Заготовки соблюдают запреты PACK-personal/CLAUDE.md §2:
- Нет дидактики: не используются «шаг», «урок», «упражнение», «неделя 1», «сначала/затем», «модуль», «попробуй».
- Нет сценария вместо метода: всё, что описано, ссылается на PD.METHOD.NNN / PD.CAT.003.
- Нет выдуманных ступеней: используется точная терминология PD.FORM.003.
