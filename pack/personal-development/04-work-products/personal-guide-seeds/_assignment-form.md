# Эталон формы: Задание с наставником (IWE)

> **Один источник формы (OwnerIntegrity).** Здесь живёт **форма** задания. **Содержание** задания — внутри каждой ступенной заготовки (`stage-N-*.md`), под bottleneck ступени. Render-скилл сопоставлений не делает: читает seed, в seed уже лежит заполненное задание.

## Происхождение

Образец-эталон — задание 1.1 из старого руководства «firefighting» («Назови систему»): опиши систему → опубликуй пост-черновик в клубе МИМ → загрузи последнюю версию FPF в чистую LLM → попроси проверить, правильно ли выделена система. Минус оригинала: **каждый раз заряжаешь чистую нейросеть спецификацией FPF заново**.

IWE снимает этот минус: наставник персистентный, личный и приватный — FPF + Pack + личная память уже встроены. Наставник помнит тебя и твой проект; конфиденциальный проект можно обсуждать с ним, а не выносить людям. Поэтому в персональном руководстве «загрузи FPF в LLM» заменяется на «поработай со своим IWE».

## Две лестницы — не путать (ArchGate 2026-06-28)

Зрелость созидателя меряется по **двум разным осям**, и руководство ориентируется на квалификацию:

- **Степень квалификации** (PD.QUAL.001, EQF L0-L8: Первокурсник → Ученик → Работник → Стратег → Специалист → Практик → Мастер → Реформатор → Общественный деятель). Присваивается **за результаты** (созданные системы, влияние на сообщество). Поле Память.Derived: `3_8_degree`. **Это основной ориентир руководства для зрелых.**
- **Учебная ступень зрелости ученика** (PD.FORM.003, 5 ступеней: Случайный → … → Проактивный). Меряет устойчивость учебного ритма *внутри* программы ЛР. Поле: `3_4_qualification.stage`. **Вторичный сигнал** — нужен, чтобы заметить провал по роли Ученика и мягко порекомендовать метод ЛР.

Тест на путаницу: «это про результаты во внешнем мире или про устойчивость моего учебного ритма?» Первое → квалификация; второе → учебная ступень.

## Гейт применимости (БЛОКИРУЮЩЕЕ для render-скилла)

Задание с наставником вставляется в руководство **только при одновременном выполнении двух условий**:

1. **Зрелость по любой из двух осей:** учебная ступень ≥4 (Дисциплинированный) **ИЛИ** степень квалификации ≥ Работник (L2). Если обе оси ниже порога — задание **не показывается** (разрыв калибра); остаётся лёгкий «вопрос для чата».
2. **IWE у пользователя реально развёрнут** (см. «Детект развёрнутости» ниже).

## Каноническая форма (5 полей)

```markdown
### Задание с наставником (IWE)

1. **Что делаешь.** <одно конкретное действие под метод/bottleneck ступени>
2. **С наставником.** Обратись к своему IWE и сформулируй запрос: «<пример запроса>».
   <!-- сноска «как именно сейчас обратиться» — подставляет render-скилл из актуального списка инструментов/скиллов IWE; в теле задания инструмент НЕ зашивается, чтобы руководство не устаревало -->
3. **Зафиксируй приватно.** Результат — в Pack / `history/` руководства. Всегда.
4. **Проверь.** Попроси наставника найти ошибку: «правильно ли я <…>?». Это и есть обученная на FPF нейросеть-наставник.
5. **Вынеси на люди (по желанию, если не конфиденциально).** Пост-черновик v0.1 в клубе МИМ с пометкой «будет v0.2». Конфиденциальный проект — пропускаешь, остаёшься на шаге 3.
```

**Почему 5 полей, а не 4.** Поля 3 и 5 разведены намеренно: приватная фиксация — всегда; публикация — по применимости. Это удерживает обе оси одновременно: приватность наставника (ради конфиденциальных проектов, за что пилот ценил формат) и социальный контур клуба (внешнее обязательство + версионирование v0.1 как анти-перфекционизм).

## Детект развёрнутости IWE (для render-скилла)

Порядок проверки — факт сначала, вопрос потом (правило IWE: проверяемый факт проверяют, а не спрашивают):

1. **Авто-проверка факта:** `github_status` (подключён ли репо `personal-guide`) + `personal_list_sources` (подключён ли личный источник через MCP). Оба сигнала положительны → IWE считается развёрнутым.
2. **Сигнал неоднозначен** (репо есть, но источник не подключён, или наоборот) → задать пользователю **один** явный вопрос: «У тебя развёрнут IWE (личный экзокортекс с подключённым MCP)?».
3. **Тир (уровень доступа) НЕ используется** как признак развёрнутости: ступень мастерства ≠ наличие развёрнутого экзокортекса. Можно быть зрелым учеником и ещё не подключить IWE.

## Если ступень ≥4, но IWE НЕ развёрнут

Вместо задания с наставником показать **задание развернуть IWE** (через `/personal-guide-start` / `/connect-guide`):

```markdown
### Задание: разверни своего наставника (IWE)

Ты дорос до работы с наставником, но IWE ещё не подключён. Первый шаг —
развернуть личный экзокортекс: `/personal-guide-start`, затем `/connect-guide`.
Как только наставник подключён — это задание сменится на работу с ним.
```

Чистая LLM + последняя версия FPF — допустима лишь как **переходная сноска-костыль** на время настройки, не как равноценная постоянная ветка:

```markdown
<!-- переходный режим: пока IWE не развёрнут — временно используй любую LLM
     с загруженной последней версией FPF (github.com/ailev/FPF). Это костыль
     на время настройки, не постоянный путь. -->
```
