# [PD.WP.020] Горизонтные документы персонального руководства

Персональное руководство собирается не из плоского набора ночных данных, а из трёх документов разных горизонтов. Каждый документ создаётся один раз в свой период и меняется один раз в период ИЛИ по событию. Документы живут в репозитории персонального руководства пользователя, каталог `horizons/` — пользователь их видит, правка файла пользователем = новая одобренная версия.

Типы данных 2.1-2.4 — по модели резидентности данных IWE (`FMT-exocortex-template/docs/DATA-RESIDENCY.md`): 2.1 — устойчивые факты о пользователе (профиль, цели, квалификация); 2.2 — оперативный поток событий (активность, здоровье, рабочее время); 2.3 — формализованное личное знание (личный Pack, Lifework-документы); 2.4 — неформализуемая личная практика (хроника решений, рефлексии).

## 1. Глобальный документ

| Характеристика | Значение |
|---|---|
| Горизонт | квартал (или реже) |
| События пересборки | получение степени квалификации Стратег (одновременно — порог подключения слоя: до Стратега документ не собирается); решение пользователя |
| Состав (из чего собирается) | характеристики личности из цифрового двойника (2.1); выжимка из формализованных личных документов — Lifework: личный устав, миссия, принципы (2.3); личный Pack, если существует (2.3); хроника практики как фон (2.4) |
| Содержание | «кто этот человек и куда он идёт»: ценности, ограничения, длинные цели, характеристики. Выжимка под руководство, не полный текст Lifework |
| Согласование | одобрение пользователя; без ответа — автопринятие по таймауту (окно длиннее, чем у младших горизонтов) |
| Вес на входе дневной генерации | фоновый (~0.1): тон, ценности, ограничения; не источник тем |

## 2. Стратегический документ

| Характеристика | Значение |
|---|---|
| Горизонт | месяц |
| События пересборки | смена степени квалификации; смена ступени Ученика; решение пользователя |
| Состав (из чего собирается) | одобренный Глобальный документ (необязателен: до его появления — сборка без глобального слоя с честной пометкой); текущая квалификация/ступень и цели месяца из цифрового двойника и стратегического контекста (2.1); **заготовка квалификации** — знание платформы «как работать с этой квалификацией» (Pack, `personal-guide-seeds/qualification-*.md`), потребляется один раз здесь и только здесь; неудовлетворённости и хроника (2.4); предыдущий Стратегический (carry-over) |
| Содержание | месячное направление: как двигается квалификация, мировоззренческая дуга, акценты программ, что считается прогрессом месяца |
| Согласование | одобрение пользователя; без ответа N дней — автопринятие |
| Вес на входе дневной генерации | направляющий (~0.3): акценты, связка тем с целями месяца, дуга квалификации |

## 3. Тактический документ

| Характеристика | Значение |
|---|---|
| Горизонт | неделя |
| События пересборки | корректировка пользователя |
| Состав (из чего собирается) | одобренный Стратегический документ (направление месяца); входные данные недели: активность, рефлексии, история руководств (2.2, 2.4); скорректированная структура прошлой недели (carry-over) |
| Содержание | недельная СТРУКТУРА, не проза: тема недели + 7 дневных слотов (тема/рычаг/программа-источник) с таргетированием на узкое место |
| Согласование | одобрение пользователя; без ответа — автопринятие по таймауту (доставка не блокируется) |
| Вес на входе дневной генерации | каркас (1.0): задаёт тему и рычаг каждого дня, обязательный |

## Инварианты

- Один документ — один горизонт — один период создания. Пересборка вне календаря — только по названному событию или решению пользователя.
- Молчание пользователя не блокирует конвейер: любой горизонтный документ принимается автоматически по таймауту своего горизонта (у старших горизонтов окно длиннее).
- Заготовка квалификации — платформенное знание Пака; в ночную генерацию напрямую не подаётся, живёт в составе Стратегического документа.
- Свежесть каждого документа видна в шапке дневного руководства (ok / stale / missing); протухший документ не блокирует доставку, но честно виден.
- Веса — калибруемые параметры конвейера, не константы знания; их актуальные значения живут в спецификации конвейера (DS-репо), здесь — только роли (каркас / направляющий / фоновый).
