# [PD.SOTA.003] Когнитивные паттерны многозадачности и параллельной оркестрации

---

## 1. Базовое ограничение: мозг не параллелит

Мозг не выполняет сознательные задачи параллельно — он переключается (task switching). Стоимость:

| Факт | Источник |
|------|---------|
| 23 мин на восстановление глубокого фокуса после прерывания | Gloria Mark, UC Irvine |
| До 40% продуктивного времени теряется на переключения | Multiple meta-analyses |
| -10 IQ при хроническом мультитаскинге | Hewlett-Packard / University of London |
| ~35 000 микрорешений/день, качество деградирует нелинейно | Baumeister (Strength Model of Self-Control) |

**Статус:** current. Это нейрофизиологическое ограничение, не культурное. ИИ его не отменяет.

**Revision criterion:** Появление нейроинтерфейсов (BCI), расширяющих рабочую память. До тех пор — current.

---

## 2. Три различения (ключевое)

| Термин | Определение | Кто работает | Эффект |
|--------|------------|-------------|--------|
| **Многозадачность** (multitasking) | Человек переключается между задачами, сам исполняет | Человек = исполнитель + диспетчер | ❌ -40% продуктивности, усталость, ошибки |
| **Когнитивная разгрузка** (cognitive offloading) | ИИ берёт рутинную когнитивную работу, человек думает глубже | Человек = мыслитель, ИИ = исполнитель | ✅ Мультипликатор, но риск когнитивного долга |
| **Параллельная оркестрация** (parallel orchestration) | N агентов работают одновременно, человек = ревьюер + диспетчер | Человек = oversight, ИИ = N исполнителей | ⚠️ Работает при структуре; ломается при N>4-5 (open-loop) |

**Ошибка смешения:** называть параллельную оркестрацию «многозадачностью» — приводит либо к ложному страху (наука против!), либо к ложной уверенности (у меня же ИИ!).

---

## 3. Парадокс ИИ-усиления

Исследования 2024-2026 фиксируют двойной эффект:

| Аспект | Активное использование (экзоскелет) | Пассивное использование (автопилот) |
|--------|-------------------------------------|-------------------------------------|
| Когнитивная нагрузка | ↓ Снижает рутинную | ↑ Информационная перегрузка (r=0.905) |
| Критическое мышление | Усиливает (глубокие диалоги) | Ослабляет (прямые ответы) |
| Автономия суждения | Сохраняется | Эрозия (r=-0.360) |
| Когнитивный долг | Минимальный | Накапливается |

**Когнитивный долг** (MIT Media Lab, 2025): ИИ позволяет пропустить когнитивную работу сейчас, но долг накапливается — аналогия с техдолгом в коде. При пассивном использовании человек теряет экспертизу в деталях.

**Bottleneck shift:** При ИИ-усилении bottleneck смещается: исполнение → oversight → decision fatigue. Мозг больше не «делает» — он «решает». Решения истощают ресурс самоконтроля (Baumeister).

**Revision criterion:** Появление ИИ-систем с верифицируемыми гарантиями качества (formal verification), снижающих oversight burden. До тех пор — current.

## 4. Параллельная оркестрация: когнитивный профиль

Паттерн: человек работает одновременно в N сессиях с ИИ-агентами. Цикл: открыть сессию → прочитать результат → дать указание → перейти к следующей.

### Нагрузка на мозг

| Компонент | Что происходит | Масштабирование |
|-----------|---------------|-----------------|
| Глубокое рассуждение | Делает агент | Не масштабируется с N |
| Оценка результата | Человек читает, решает: ок / не ок | Линейно с N × возвратов |
| Формулировка указания | Короткое направление | Линейно с N |
| Context warm-up | 2-3 мин при входе в сессию (с WP-context) | Линейно с N |
| Decision fatigue | Каждый возврат = решение | **Нелинейно** — качество падает |

### Стоимость переключения: классика vs экзоскелет

| Режим | Стоимость одного переключения | Источник |
|-------|------------------------------|---------|
| Без системы | ~23 мин (мозг реконструирует контекст) | Gloria Mark |
| С ОРЗ + WP-context | ~2-3 мин (экзоскелет грузит контекст) | Эмпирика IWE |
| Глубокая задача (problem-framing) | ~23 мин даже с экзоскелетом | Warm-up рассуждения не сводится к фактам |

### Пороги безопасности

| Класс задач | Безопасный N параллельных | Почему |
|-------------|--------------------------|-------|
| trivial / closed-loop | ≤6 | Ревью поверхностное, агент верифицирует себя (тесты) |
| open-loop | ≤4 | Ревью требует суждения, decision fatigue растёт |
| problem-framing | 1 | Глубокое рассуждение не дробится |

**Revision criterion:** Эмпирическая валидация порогов через decision quality tracking в IWE (когда будет реализован). До тех пор — hypothesis, основанная на литературе + наблюдении.

---

### Эмпирика практика: траектория освоения мультисессионной работы (не научный источник)

**Статус источника:** личное наблюдение одного практика (не выборка, не исследование) — дополняет §4, не заменяет научные метрики выше.

Мультисессионная работа с несколькими параллельными ИИ-агентами требует не только системного мышления, но и физической выносливости. По наблюдению практика с 6-летним опытом: переход от 3-4 часов до устойчивой 8-часовой работы с ИИ-агентами занял 6 лет практики. Без регулярных перерывов интенсивная работа продолжительностью 2-3 часа может «сжечь» не только дневной, но и недельный запас энергии.

**Как читать это наблюдение:** оно не отменяет пороги безопасности §4 (N≤4-5 для open-loop) — оно указывает, что даже при соблюдении структурных порогов сама продолжительность непрерывной мультисессионной работы имеет отдельную кривую освоения, измеряемую годами, а не неделями.

## 5. Шесть принципов безопасной оркестрации (из исследований 2025-2026)

1. **Cap concurrency** — ограничить число параллельных агентов, запрашивающих ревью
2. **Rhythm-driven review** — батчить некритичные результаты в структурированные окна
3. **Metadata на эскалации** — confidence, provenance, reversibility на каждом выходе агента
4. **Explicit thresholds** — формализовать когда агент действует автономно vs эскалирует
5. **Recovery rights** — защищённое время без эскалаций (= перерывы помидорок)
6. **Sustainability metrics** — трекать плотность решений и утомление, не только throughput

Источник: Springer (Overloaded minds and machines, 2026), Medium (Human Oversight Under Load, 2026).

## 6. Формула мультипликатора

```
Мультипликатор = f(структура_работы, качество_handoff)
```

НЕ `f(количество_параллельных_задач)`.

Мультипликатор 2.68x (W14) достигнут через:
- ОРЗ-фрактал (структура)
- WP-context files (handoff)
- Когнитивная разгрузка (экзоскелет), не многозадачность

При росте N параллельных сессий мультипликатор растёт до порога, затем падает из-за decision fatigue.

## 7. Связанные failure modes

| FM | Описание | Детекция |
|----|---------|---------|
| **Ложная многозадачность** | Называть оркестрацию «многозадачностью» → ложный страх или ложная уверенность | Терминология в речи/документах |
| **Когнитивный долг** | Пассивное использование ИИ → потеря экспертизы | Снижение качества ревью при стабильном throughput |
| **Decision fatigue скрытая** | Микрорешения копятся, качество падает незаметно | Нет метрик → нет детекции (gap) |
| **Oversight degradation** | Глубина ревью падает с ростом N | Формальное «ок» без содержательной проверки |
