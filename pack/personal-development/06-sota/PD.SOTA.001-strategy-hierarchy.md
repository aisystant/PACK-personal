# [PD.SOTA.001] Strategy Hierarchy

---

## Иерархия уровней стратегирования

Четыре уровня стратегирования и три операционных уровня исполнения:

```
                    S-НЭП (PD.WP.005 Dissatisfactions)
                    поперёк всей вертикали — источник смысла
                    │
МИССИЯ              ├── S1, S3, S8
                    │
ВИДЕНИЕ (год)       ├── S1, S2, S6, S11 ◄── Калибр (горизонт мышления)
                    │
R МЕСЯЦА            ├── S1, S4...       ◄── ТОС (где constraint месяца?)
                    │
WEEKPLAN            │                  ◄── MAPSTRATEGIC (фазы систем)
                    │
DAYPLAN             │
                    │
РП (WP-NNN)         ├── O1–O4
                    │
SESSION             │
                    │
                    O-НЭП (операционные, создают РП однажды при фиксации)
```

> **Решение (2026-04-01):** квартальный уровень (Q-цели) убран. Горизонты: Год (видение + желания) → Месяц (R-результаты) → Неделя → День. Квартал создавал лишний слой без ощутимой пользы: месяц достаточно короток для оперативной коррекции, год достаточно длинен для направления.

### Описание уровней

| Уровень | Горизонт | Документ | Обновляется |
|---------|----------|----------|-------------|
| Миссия | 10+ лет | Strategy.md §Миссия | Редко (смена смысла) |
| Видение / Год | 1–7 лет | Strategy.md §Год | Ежегодно (Annual Session) |
| R месяца | Месяц | Strategy.md §Результаты месяца | Первая Strategy Session месяца |
| WeekPlan | Неделя | WeekPlan W{N}.md | Каждый понедельник (Strategy Session) |
| DayPlan | День | DayPlan YYYY-MM-DD.md | Каждое утро (Day Open) |
| РП (WP) | Недели–месяцы | WORKPLAN.md (в каждом репо) | При создании и закрытии |
| Session | Часы | WP-context file | Перед каждой рабочей сессией |

## Сквозные инструменты

Три инструмента не принадлежат одному уровню — они входят поперёк вертикали:

| Инструмент | Уровень входа | Что делает |
|-----------|--------------|-----------|
| **Калибр** (PD.CHR.007) | Видение/Год + R месяца | Задаёт горизонт мышления при постановке целей; показывает что нужно для следующего уровня |
| **ТОС-месяца** | R месяца | Находит единственный constraint, который блокирует все нижележащие результаты |
| **MAPSTRATEGIC** | WeekPlan | Показывает текущую фазу каждой системы при агрегации РП Стратегом |

**НЭП** — не инструмент, а источник. Стратегические (S) питают верхние уровни (Миссия→Год). Операционные (O) создают конкретные РП.

## Синтез SOTA-методов

| SOTA-метод | Что берём | Где в иерархии |
|-----------|-----------|---------------|
| **RBM** (Results-Based Management) | Цепь Impact→Outcomes→Outputs→Activities→Inputs | Вся вертикаль: Миссия=Impact, Год=Outcomes, R=Outputs, РП=Activities, h=Inputs |
| **OKR** | Objective (направление) ≠ Key Result (измеримый) ≠ Initiative (работа) | Год = Objectives, R = Key Results, РП = Initiatives |
| **TOC** (Theory of Constraints) | Один constraint на уровень; фокус на узком месте | ТОС-месяца на уровне R; ТОС-недели на уровне WeekPlan |
| **Shape Up** | Appetite (бюджет h) → Bet (результат) → Scope (работа) | R месяца = Bet с appetite; РП = Scope |
| **JTBD** | Jobs (почему) отделены от Solutions (что делать) | S-НЭП = Jobs; Год и R = решения Jobs |

## Ключевые различения

| Различение | Почему важно |
|-----------|-------------|
| Стратегирование ≠ Планирование (D.017) | Стратегирование выбирает ЧТО; планирование распределяет КОГДА и КАК |
| S-НЭП ≠ O-НЭП | Стратегические — источник смысла (редко меняются). Операционные — создают конкретный РП (часто меняются) |
| Видение года (Objectives) ≠ R месяца (Key Results) | Год — желаемые состояния и направление. R — измеримый результат этого месяца, продвигающий год |
| Квартал — не уровень иерархии | Квартал создаёт лишний слой без пользы: месяц достаточно короток для коррекции, год достаточно длинен для направления |
| РП (Initiative) ≠ Задача | РП — артефакт с критериями done и бюджетом. Задача — действие без критерия |
| ТОС-месяца ≠ приоритет | ТОС — то, что блокирует ВСЁ остальное. Приоритет — важность при прочих равных |

## Связи между уровнями (что должно быть явным)

```
Год (желания)  ←──  R месяца (колонка → желание года)
R месяца       ←──  НЭП (колонка НЭП)
R месяца       ←──  РП (таблица РП → Результаты в Strategy.md)
WeekPlan       ←──  MAPSTRATEGIC (агрегация фаз систем)
РП             ←──  O-НЭП (при создании РП указывается операционная НЭП)
```

## Revision Criterion

Статус изменится если:
- Появится метод, который интегрирует все пять в единую формализованную систему с доказанной эффективностью для индивидуальных стратегов
- Практика покажет, что один из пяти методов создаёт противоречие с другими в контексте IWE
