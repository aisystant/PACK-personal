# [PD.SOTA.006] Адаптивная персонализация в обучении

---

## 1. Назначение

Обзор state-of-the-art методов адаптивной персонализации: не литобзор — **карта применения** к программе ЛР. Для каждого метода: что это, доказательная база, как влияет на каскад планирования (METHOD.025).

---

## 2. BKT — Bayesian Knowledge Tracing

### Суть

Вероятностная модель освоения навыка, предложенная Corbett & Anderson (1994). Четыре параметра:

| Параметр | Обозначение | Типичное значение |
|----------|------------|-------------------|
| Начальная вероятность освоения | P(L₀) | 0.0 – 0.3 |
| Вероятность угадывания (знает ответ, но навык не освоен) | P(G) | 0.05 – 0.3 |
| Вероятность ошибки при освоенном навыке | P(S) | 0.05 – 0.2 |
| Вероятность освоения после одной практики | P(T) | 0.2 – 0.5 |

После каждой практики модель обновляет P(L) по теореме Байеса. Навык «освоен» при P(L) ≥ 0.95 (порог mastery criterion, конфигурируется).

### Доказательная база

- **ASSISTments (2004–2023):** 2.7 млн учеников, снижение времени до mastery ~20% по сравнению с равномерной практикой (Heffernan & Heffernan, 2014).
- **Intelligent Tutoring Systems:** мета-анализ VanLehn (2011): ITS с BKT — effect size +0.76 SD по сравнению с обычным классом.
- **Ограничение:** BKT предполагает независимость навыков. Для методов саморазвития (M1/M2/M4) навыки сильно коррелируют → чистый BKT не применим напрямую.

### Применение к программе ЛР

BKT в чистом виде не используется — методы ЛР не сводятся к «ответил правильно / неправильно». Однако принцип **накопительного апостериорного обновления** применим:

- Диагност собирает сигналы (слоты W/M1/M2/M4/IT/A из RCS) — аналог «попыток»
- Каждый сигнал (артефакт, коммит, заполненный слот) обновляет оценку baseline по FORM.089 §6.1
- Принцип mastery criterion: переход к следующей ступени только при P(L) ≥ threshold для всех bottleneck-слотов

**Ключевой инсайт для каскада:** BKT обосновывает **не регенерировать план каждый день** — накопленная байесовская оценка достоверна только при нескольких сигналах. Дневная корректировка (тактика) — это не переоценка модели, а исполнение гипотезы недели.

## 3. IRT — Item Response Theory

### Суть

Психометрическая теория (Rasch, 1960; Birnbaum, 1968), описывающая вероятность правильного ответа через соотношение сложности задания и способности учащегося.

Трёхпараметрическая модель (3PL):

```
P(правильный ответ) = c + (1-c) / (1 + exp(-a(θ - b)))
```

- θ = способность учащегося
- b = трудность задания
- a = дискриминативность задания (насколько чётко разделяет уровни)
- c = вероятность угадывания

### Доказательная база

- Базис большинства адаптивных тестов: SAT, GRE, GMAT используют IRT с 1990-х.
- CAT (Computerized Adaptive Testing): сокращает количество вопросов для оценки уровня в 2–3 раза при сохранении точности (Wainer, 2000).
- Российский контекст: ЕГЭ использует элементы IRT (шкалирование).

### Применение к программе ЛР

IRT обосновывает алгоритм диагностики (FORM.089 §6.1):

- **Якорные вопросы Диагноста** (фаза 1) — аналог высокодискриминативных заданий (a >> 1): быстро разделяют ступени 2–3–4 без исчерпывающего опроса
- **Уточняющие вопросы** (фаза 2) — зонирование bottleneck слота (θ ≈ b для целевого слота)
- **≤5 вопросов** — IRT CAT-принцип: достаточно для оценки θ с SE < 0.3 при правильном подборе заданий

**Ключевой инсайт для каскада:** IRT обосновывает, почему диагностика ступени проводится **раз в квартал**, а не еженедельно: после 5 точных вопросов θ оценена стабильно; частые переизмерения не увеличивают точность (SE плато), но создают quantum-like эффект (HD #50, DP.SOTA.020) — наблюдение влияет на состояние.

## 4. Spaced Repetition (интервальные повторения)

### Суть

Эффект интервалов — материал лучше запоминается при повторении через возрастающие промежутки времени (Ebbinghaus, 1885; Bahrick et al., 1993).

Алгоритмы:

| Алгоритм | Год | Ключевое правило | Доказательная база |
|----------|-----|-----------------|-------------------|
| **SuperMemo SM-2** | 1987 | Интервал умножается на EF (2.5 по умолчанию) при правильном ответе | >1000 пользователей, Wozniak 1990 |
| **SM-18** | 2020 | Нейросетевая оптимизация EF под каждого пользователя | Нет публикации, но Anki использует SM-2 (90M+ пользователей) |
| **FSRS** | 2022 | Free Spaced Repetition Scheduler, открытый, 4 параметра (stability, difficulty, retrievability, forgetting index) | Validation на 20k+ пользователях Anki, RMSE < SM-2 |
| **HLR (Half-Life Regression)** | 2016 | Duolingo, оценивает half-life через историю ответов | Settles & Meeder, NAACL 2016 |

**Кривая забывания:** без повторения через 1 день забывается ~40%, через неделю — ~65%, через месяц — ~80% (Ebbinghaus, нормализованные оценки).

### Применение к программе ЛР

Методы ЛР — не карточки, но принцип «интервальных касаний» применим:

- **Месячный цикл тем** (каскад: месяц → 1–3 темы) — обоснование: одно касание метода недостаточно для перехода к автоматизму (HD «Автоматизм ≠ Внимание»). Тема возвращается через 3–4 недели с новым контекстом
- **Еженедельная гипотеза роста** — практика метода с нарастающей сложностью (EF-аналог: сложность задания растёт при успехе предыдущей недели)
- **Ежедневные тактические ДЗ** — короткие касания (помидорки) поверх недельной гипотезы: закрепление, не изучение нового

**Ключевой инсайт для каскада:** Spaced Repetition обосновывает **трёхуровневость** (квартал / месяц / неделя) вместо «недельного плана от начала». Без месячного горизонта метод изучается один раз и не закрепляется. Квартал = 3 «волны» месячных касаний = устойчивый переход к автоматизму.

## 5. Curriculum Learning — порядок подачи материала

### Суть

Bengio et al. (2009) предложили тренировать нейронные сети, подавая данные от простых к сложным — по аналогии с тем, как учатся дети. Ключевые принципы:

1. **Easy-first:** начало с освоенных понятий снижает тревожность и создаёт ментальные крюки для новых
2. **Scaffolding:** временная поддержка (подсказки, упрощения) убирается по мере освоения (Wood, Bruner & Ross, 1976)
3. **Prerequisite ordering:** зависимости между темами определяют порядок подачи
4. **Self-paced curriculum:** ученик сам определяет темп (Самоуправляемое обучение, Knowles, 1975)

### Доказательная база

- Последовательное изучение с нарастающей сложностью улучшает retention: мета-анализ Kornell & Bjork (2008), effect size +0.28 по сравнению с случайным порядком.
- **Desirable Difficulties** (Bjork, 1994): намеренное усложнение (интерливинг, тест перед обучением) снижает краткосрочное выполнение, но улучшает долгосрочное запоминание.
- **Worked examples → problem solving:** постепенный переход от примеров к самостоятельному решению (Kalyuga et al., 2003, expertise reversal effect).

### Применение к программе ЛР

- **Ступени 1–5 как curriculum:** порядок M1 → M2 → M4 определён структурой зависимостей (нельзя освоить IWE/ОРЗ без базовой собранности M1). FORM.080 §3.1 кодирует prerequisite ordering
- **Мировоззренческая дуга как scaffold:** «Я могу меняться» → «Я — система» → «Мир — система» — постепенное снятие опоры (дуга нарратива Навигатора, role-prefixes.md)
- **Variant B гипотезы недели:** тактика дня = scaffolding гипотезы недели (подсказки, разбор, корректировка), а не самостоятельный план

**Ключевой инсайт для каскада:** Curriculum Learning обосновывает **фиксированность программного горизонта**: нельзя переставлять ступени под запрос ученика, не нарушая prerequisite ordering. Квартальный и месячный горизонты — адаптивны (что изучаем, как); программный — фиксирован (в какой последовательности).

## 6. Ключевые инсайты для программы ЛР

### Как SOTA влияет на каскад планирования (METHOD.025)

| Горизонт | Алгоритм | Инсайт |
|----------|----------|--------|
| **Программа (фикс.)** | Curriculum Learning | Prerequisite ordering ступеней не нарушаем |
| **Квартал** | IRT | Диагностика ступени ≤5 вопросов, раз в квартал (не чаще) |
| **Месяц** | Spaced Repetition | 1–3 темы × 3–4 недели = одна «волна» закрепления метода |
| **Неделя** | BKT + SR | Гипотеза роста = недельная «попытка» по байесовской оценке bottleneck |
| **День** | Curriculum (scaffold) | Тактика = scaffold гипотезы недели, не самостоятельный план |

### Что из SOTA НЕ применимо напрямую

- **Чистый BKT:** требует бинарных «правильно/неправильно» сигналов. Для методов ЛР — приближение через слоты RCS (1–5 по каждому слоту).
- **Anki-подобные карточки для методов M1/M4:** методы осваиваются через практику, не воспроизведение. Карточки подходят для терминологии, не для навыков.
- **CAT в реальном времени:** оценка θ обновляется после каждого вопроса — для ЛР это quantum-like эффект (HD #50), частые измерения искажают поведение. Правильно: редкие baseline-проверки на Month Close.

### Variant B как SOTA-решение

Variant B (гипотеза роста недели → тактика дня) согласован со всеми четырьмя алгоритмами:

- BKT: неделя = накопление сигналов, байесовское обновление не сразу
- IRT: дневная оценка слишком нестабильна (SE высокое), недельная достаточна
- SR: дневные касания поверх недельной темы = оптимальные интервалы
- CL: scaffold гипотезы недели = desirable difficulty без перегрузки
