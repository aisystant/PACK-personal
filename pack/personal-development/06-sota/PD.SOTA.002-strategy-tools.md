# [PD.SOTA.002] Strategy Tools

---

## Сравнение методов: что взять в IWE

### Методы управления задачами и информацией

| Метод | Суть | Сильная сторона | Слабая сторона | Взято в IWE |
|-------|------|-----------------|----------------|-------------|
| **GTD** (David Allen) | Capture → Clarify → Organize → Reflect → Engage | Нулевой inbox; 2-минутное правило; разделение сбора и обработки | Не отвечает на «зачем»; не помогает выбирать между проектами | 2-минутное правило; принцип «capture first, process later» (= fleeting-notes) |
| **PARA** (Tiago Forte) | Projects / Areas / Resources / Archives | Простая иерархия для цифровых артефактов; работает поверх любого инструмента | Не про задачи, не про стратегию; легко деградирует в архив | Структура DS-репо (governance/instrument/surface); принцип «один факт — одно место» |
| **Zettelkasten** (Luhmann) | Атомарные заметки + двусторонние ссылки = сеть идей | Долгосрочное накопление мысли; органическое открытие связей | Высокий порог входа; избыточен для операционного планирования | НЕ берём как операционный метод. Принцип атомарности — в captures.md (одна заметка = одна идея) |
| **Second Brain** (BASB) | CODE: Capture → Organize → Distill → Express | Связывает чтение/заметки с публикацией; progressive summarization | Размытые границы между уровнями; легко утонуть в организации | Творческий pipeline: fleeting → draft → публикация (PD.FORM.005) |
| **OKR** | Objective (направление) + Key Results (измеримые) + Initiatives | Разделяет «куда» и «как измерить»; фокус на малом числе приоритетов | Стандартный каденс — квартал; создаёт лишний уровень для личного использования | Год = Objectives, R месяца = Key Results, РП = Initiatives. **Квартал убран** — достаточно год+месяц |
| **Shape Up** (Basecamp) | Appetite → Bet → Scope (6 недель) | Тайм-боксирование вместо оценок; cooldown после цикла | Заточен под команды; 6 недель — не всегда подходящий горизонт | Appetite (бюджет h) → Bet (R месяца) → Scope (РП). Cooldown → Strategy Session |
| **TOC** (Goldratt) | Найти единственный constraint → фокусировать на нём | Объясняет почему распыление убивает прогресс | Требует честной диагностики; легко выбрать неверный constraint | ТОС-месяца: один constraint, который блокирует все R. Полностью взят |

### Ключевое различение между методами

| Вопрос | Метод |
|--------|-------|
| **Куда идти?** (стратегия) | OKR (Objective), JTBD (Job-to-be-Done), НЭП |
| **Как измерить?** | OKR (Key Results), RBM (Outputs) |
| **Что делать?** (планирование) | GTD, Shape Up (Scope), РП |
| **Как организовать информацию?** | PARA, Zettelkasten, Second Brain |
| **Где узкое место?** | TOC |

IWE покрывает все пять вопросов. Ошибка — использовать инструмент не для своего вопроса (например, Zettelkasten для оперативного планирования).

## Инструменты 2025-2026: что происходит на рынке

| Инструмент | Философия | Статус 2026 | Применимость к IWE |
|-----------|-----------|------------|-------------------|
| **Obsidian** | Локальные файлы + граф связей. «Ваши заметки — ваши файлы навсегда» | Бесплатен (включая коммерческое использование с фев 2025). Sync $4/мес. Быстро растёт экосистема AI-агентов | **Используется** (DS-my-strategy, fleeting-notes). Подходит: локальный контроль, Markdown, git-совместим |
| **Notion** | Collaborative databases. AI-агент (Notion 3.0, сен 2025) | Notion 3.0 — полноценный AI-агент, переписан с нуля. Лучший для команд | НЕ используется. Избыточен для личного IWE; данные в облаке Notion |
| **Roam Research** | Bidirectional linking, block-based | $165/год. Сдаёт позиции — Obsidian закрывает те же задачи дешевле | НЕ используется |
| **Logseq** | Outline-first + bidirectional links + local-first | Активно развивается. Гибрид Obsidian + Roam, бесплатный | Альтернатива Obsidian, но не даёт преимуществ над текущим стеком |
| **Claude Code / AI-агенты** | Агент работает прямо в репозитории / vault | 2026: AI-агенты интегрируются в Obsidian vault напрямую. Claude Code = текущий стек IWE | **Это и есть IWE**. Агент читает/пишет MD-файлы в git-репо |

### Вывод по инструментам

**Текущий стек IWE оптимален для 2026:**
- Markdown + git = долгосрочная сохранность, версионирование, локальный контроль
- Claude Code = AI-агент работает с теми же файлами без экспорта/импорта
- Obsidian = опциональный просмотрщик (без синхронизации с git-репо не нужен)

**Что НЕ менять:** инструмент не создаёт систему. Переход на Notion/Roam/Logseq не решит ни один из разрывов Р1–Р8 — они про ритуалы и маршрутизацию, не про UI.

## Сравнение каденсов стратегирования

| Подход | Каденс | Практика 2026 |
|--------|--------|--------------|
| OKR-стандарт | Год → Квартал → Месяц | Большинство компаний. Квартал = балансировочный уровень |
| **IWE (решение 2026-04-01)** | **Год → Месяц → Неделя → День** | Квартал убран. Достаточно: год даёт направление, месяц — оперативную коррекцию |
| Shape Up | 6 недель + cooldown | Хорошо для продуктовых команд; в личном использовании cooldown = Strategy Session |
| GTD | Ежедневный + еженедельный review | Нет горизонта выше недели — требует дополнения стратегическим уровнем |

**Почему квартал не нужен в личном стратегировании:**
1. Личная система может корректироваться ежемесячно без накладных расходов
2. Квартал создаёт буфер между ощущением (НЭП) и действием (R месяца)
3. В корпоративном OKR квартал нужен для синхронизации команд — в личном использовании этой потребности нет

## Revision Criterion

Статус изменится если:
- Появится инструмент, который объединяет локальный Markdown + git + AI-агент + визуализацию связей без дополнительной оркестровки
- Практика покажет, что отсутствие квартального горизонта создаёт разрыв между годовыми желаниями и месячными результатами
