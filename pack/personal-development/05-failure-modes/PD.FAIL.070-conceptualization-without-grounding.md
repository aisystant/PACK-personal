# [PD.FAIL.070] Conceptualization Without Grounding

---

## Definition

Систематический паттерн обучения, при котором освоение понятия останавливается на одном из ранних уровней (0–1 по PD.FORM.101 §3) — цитирование определения, узнавание термина в чужих текстах — без перехода к применению в собственной материальной ситуации. Понятие функционирует как **ярлык** (метка, которую можно прикрепить), а не как **инструмент** (то, чем можно работать).

**Ключевой симптом**: правильные слова при отсутствии соответствующих действий. Человек уверенно говорит о «защищённом слоте», «системном мышлении», «мировоззрении как коде» — но в его недельном расписании, заметках, артефактах эти понятия не оставляют следа.

**Источник режима отказа**: цикл моделирования (PD.FORM.102 §4) состоит из двух операций — концептуализации (от частного к общему) и заземления (от общего к частному). Здоровое освоение — полный цикл. Этот режим — обрыв на верхней дуге: концептуализация продолжается, заземление не выполняется.

---

## Error Type

**Type**: `epistemological` — системная ошибка в способе перехода от знакомства с понятием к его освоению. Не дефект памяти и не нехватка усилий — нарушение нормы цикла моделирования.

---

## Detection Test

| Симптом | Пример |
|---------|--------|
| Цитирует определение, не может привести свой пример | «Защищённый слот — это…» — длинное определение; «у тебя был защищённый слот на этой неделе?» — пауза, обтекаемый ответ |
| Применяет к чужим кейсам, не к своим | Разбирает понятие на кейсе пилота, не может перенести на свою работу |
| Длинные обзоры без артефактов | Прочитал 10 книг по системному мышлению, в журнале саморазвития нет ни одного применения |
| Понятие как украшение речи | Использует термин в обсуждениях, но за термином не стоит наблюдаемое действие |
| «Понял, но пока не применил» — устойчивая позиция | Период «между пониманием и применением» растягивается на месяцы и не закрывается |
| Распознаёт ошибку у других, не у себя | Видит концептуализацию без заземления у других; собственные понятия считает применёнными |
| Отсутствует уровень 3 для ключевых понятий программы | По PD.FORM.101 §3 — ни одного понятия на уровне «изменение действий через понятие» |

**Тест-вопрос**: «Назови понятие из последних 4 недель обучения и его материальные следы в твоей работе — артефакт, действие, признак.» Если ответа нет или ответ типа «я понимаю это, я об этом думаю» — режим отказа активен.

## Why This Is an Error

| Концептуализация без заземления | Полный цикл моделирования |
|----|----|
| Знание = умение процитировать | Знание = умение применить в новой ситуации |
| Понятие проверяется тестом на знание определения | Понятие проверяется тестом на изменение действий |
| Накапливаются ярлыки | Накапливается мастерство |
| Уверенность в освоении на основе чтения | Уверенность в освоении на основе следов в работе |
| Тест применимости к незнакомой ситуации не проводится | Тест применимости — обязательная часть освоения |

**Глубинная проблема**: оставаясь на уровне описаний, мышление не несёт ответственности перед физическим миром (см. D.159, D.158). Нет проверки → нет коррекции → нет уточнения модели. Мышление становится **самозамкнутым языком**, который выглядит точным и при этом не способен делать работу в материальной реальности.

## Root Causes

| Причина | Описание |
|---------|----------|
| **Имитация компетентности дешевле освоения** | Цитата понятия даёт быстрый социальный сигнал «я разбираюсь». Заземление требует времени и риска (попытка может провалиться) |
| **Среда обучения работает на тестах знания** | Курсы, экзамены, статьи проверяют узнавание и формулирование, а не применение. Подкрепляется поведение «выучил термины» |
| **Отсутствие явной нормы заземления** | Без формализации заземления (PD.FORM.101) и метода (PD.METHOD.034) человек не знает, что освоение требует отдельной операции, и считает прочтение освоением |
| **Меметика «понимание = знание»** | Распространённое убеждение, что если человек понял идею, он её освоил. Меметика разрывает связь «понимание ↔ применение» |
| **Боязнь профанации** | «Применю — упрощу, исказу, не выйдет красиво» → не применяет. Перфекционизм мешает заземлению |
| **Отсутствие обратной связи от среды** | Среда не возвращает «ты использовал это понятие неправильно» — потому что использования вообще нет |

## Risk / Harm

| Риск | Описание |
|------|----------|
| **Прогресс программы ЛР останавливается** | Без перехода понятий в действия ступень роста не меняется, диагностические индикаторы (R28) не растут |
| **Иллюзия мастерства** | Самооценка опирается на знание, а не на работу — расхождение с объективной картиной растёт, корректировка откладывается |
| **Эрозия словаря** | Понятия, не подкреплённые заземлением, постепенно вытесняются другими — рабочий язык не формируется |
| **Передача невозможна** | Чтобы научить другого, нужно показать применение. Без заземления — передавать нечего, кроме цитат |
| **Накопление «образовательного шума»** | Прочитано много, в работе не отразилось → ощущение «я постоянно учусь, но не двигаюсь» |
| **Разрыв системного мировоззрения** | Мировоззрение (PD.FORM.097) — частный случай модели; без заземления остаётся декларацией |

## Correction

1. **Применить вопрос-якорь (PD.METHOD.035)**: для каждого нового понятия — «описание чего это?». Без ответа на уровне материального объекта — не двигаться дальше.
2. **Зафиксировать три опоры (PD.METHOD.034)**: материальный объект, наблюдаемое действие, проверяемый признак. Все три обязательны.
3. **Сократить ввод, увеличить применение**: вместо 10 новых понятий в неделю — 2 понятия с заземлением каждое. Темп освоения = темп заземления, не темп чтения.
4. **Журнал заземлений**: один раз в неделю — список понятий и их следов в работе за неделю. За 4 недели становится виден разрыв «прочитал ↔ применил». Закрытие разрыва — следующий приоритет.
5. **Тест на новой ситуации**: каждое ключевое понятие применить к ситуации, не использовавшейся при освоении. Не получается — модель не достроена в части заземления.
6. **Снизить порог применения**: первое применение нового понятия не должно быть «правильным» — должно быть **первым**. Уточнение качества — на следующих итерациях.
