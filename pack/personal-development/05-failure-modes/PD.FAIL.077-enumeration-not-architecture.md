# [PD.FAIL.077] Перечисление методов вместо архитектурной концепции

---

## Признак

Архитектурный концептуальный документ (concept-*.md) содержит раздел-список компонентов (методов, элементов, модулей) вместо описания структуры и принципов.

**Конкретный случай:** concept-lr-architecture.md §5 «Восемь методов» = список M001–M008. Методы уже формализованы в PACK-personal PD.METHOD.* — раздел дублировал Pack, а не раскрывал архитектуру.

## Когда возникает

При написании концепции «от содержания» (что программа содержит), а не «от структуры» (почему так организовано). Автор знает компоненты и перечисляет их — это кажется описанием архитектуры, но не является им.

## Симптомы

- Раздел называется «Методы», «Элементы», «Компоненты», «Список...»
- Содержание = bullet list или нумерованный список артефактов
- Раздел не отвечает: «почему именно так?», «какой принцип?», «какой разрыв закрывает?»
- Удаление раздела не меняет понимание концепции (информационно пуст для архитектуры)

## Detection Test

Вопрос к разделу: «Что я узнаю об архитектуре из этого раздела, чего не знал бы, посмотрев список компонентов в Pack?»

Если ответ «ничего» — PD.FAIL.077.

## Коррекция

Архитектурная концепция отвечает на:
- **«Почему такая структура?»** → принципы, разрывы, интерфейсы
- **«Что содержит?»** → одна строка-ссылка на Pack

Подробности компонентов — в Pack (OwnerIntegrity: один факт — одно место).
