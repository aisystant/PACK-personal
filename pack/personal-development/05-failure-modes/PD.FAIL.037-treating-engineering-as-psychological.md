# Антипаттерн: Психологизация инженерной проблемы (Treating Engineering Problem as Psychological)

## 1. Описание

Агент описывает трудности исключительно в психологических терминах («я ленивый», «у меня травма», «у меня комплекс») вместо инженерных («у меня нет практики декомпозиции задач», «у меня нет цикла пересмотра стратегии»).

## 2. Симптомы

- Использование слов «ленивый», «нет мотивации», «слабый характер» для описания проблем
- Бесконечный самоанализ без системного улучшения
- Обращение к психологу по вопросам, которые решаются инженерными методами

## 3. Корневая причина

Агент не владеет различением D.086 (Психологический язык vs. Инженерный язык) и использует единственный доступный фреймворк — психологический.

## 4. Коррекция

Перевести каждую «психологическую» жалобу в инженерный язык: какой системы, функции или навыка не хватает?
- «Я ленивый» → «У меня нет защищённого слота для этого проекта»
- «Нет мотивации» → «Нет связи между задачей и неудовлетворённостью»
- «Нет дисциплины» → «Нет инфраструктуры (D.010)»
