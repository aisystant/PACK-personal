# Формализация: Компиляция как строительство нейромаршрутов (Compilation as Neural Route Building)

## 1. Определение

Компиляция знания — процесс преобразования декларативного знания (описаний, артефактов) в процедурное знание (мировоззрение, навыки), встроенное в агента. Обратный процесс — де-компиляция — извлечение tacit knowledge из мировоззрения агента в отчуждаемую форму (описания).

Аналогия из CS: описание (Pack) = исходный код, мировоззрение = скомпилированная программа, обучение = компилятор.

## 2. Три направления

| Направление | Из → В | Механизм | Потери / особенности | Пример |
|-------------|--------|----------|---------------------|--------|
| **Компиляция** | Описание → Мировоззрение | Обучение, практика, интернализация | Добавляется контекст агента, результат уникален | Студент учит Pack, практикует, «видит» решения |
| **Де-компиляция** | Мировоззрение → Описание | Рефлексия, формализация, экстракция | Теряется воплощённое, контекстное, соматическое | Эксперт пишет книгу, Knowledge Extractor извлекает паттерны |
| **Открытие** | Мировоззрение → **Новое** знание | Инсайт через призму скомпилированного опыта, затем де-компиляция в описание | Порождение новых категорий мышления, а не извлечение существующих | Менделеев увидел периодическую таблицу; AlphaFold нашёл структуры белков (рекомбинация) |

Де-компиляция всегда **с потерями** (Поланьи, 1966: «we know more than we can tell»). Открытие — обратный процесс: сначала мировоззрение порождает новое видение, затем оно де-компилируется в отчуждаемое описание. AI способен к **рекомбинации** (находить неожиданные связи в данных), но пока не к **порождению** новых категорий мышления — это открытый вопрос (Q.009 в Strategy.md).

## 3. Соответствия на нейрональном уровне

| Когнитивный уровень | Нейрональный механизм | Современные данные |
|---------------------|-----------------------|-------------------|
| Знание | Активация нового нейронного соединения | Nelli et al. (2023, Neuron): люди способны к few-shot реорганизации нейронных кодов |
| Повторение | Хеббовское усиление синапса | Neuroscience & Biobehavioral Reviews (2024): синаптическая пластичность объясняет распределение, новизну, предварительное знание |
| Компиляция | Реструктуризация синаптических весов (часто используемое → автоматическое) | Anderson ACT-R (1993, 2004): declarative → knowledge compilation → procedural |
| Мировоззрение | Оптимизированная маршрутизация сигналов, работающая в условиях неопределённости | Nature (2025): мозг переиспользует когнитивные «блоки», рекомбинируя их для новых паттернов |
| Мышление письмом | Активное усиление сигнала и создание маршрутов | — |

## 4. Модель ACT-R: три стадии компиляции (Anderson, 1983, 1993, 2004)

| Стадия | Характеристика | Тип памяти | Аналогия |
|--------|---------------|------------|----------|
| **Declarative** | Инструкции в рабочей памяти, шаги выполняются последовательно | Декларативная | Интерпретатор (медленно, но гибко) |
| **Knowledge compilation** | Composition + proceduralization | Переходная | JIT-компиляция |
| **Procedural** | Автоматическое выполнение без обращения к декларативной памяти | Процедурная | Скомпилированная программа (быстро, но менее гибко) |

Два механизма компиляции: **composition** (несколько правил → одно) и **procedural strengthening** (укрепление через повторение).

## 5. Модель SECI (Nonaka & Takeuchi, 1995): маппинг

| SECI-режим | Из → В | В терминах компиляции |
|------------|--------|----------------------|
| Socialization | Tacit → Tacit | Передача мировоззрения через совместную работу (без описаний) |
| **Externalization** | **Tacit → Explicit** | **Де-компиляция: мировоззрение → Pack** |
| Combination | Explicit → Explicit | Систематизация внутри Pack |
| **Internalization** | **Explicit → Tacit** | **Компиляция: Pack → мировоззрение** |

## 6. Модель Дрейфуса: стадии компиляции (Dreyfus & Dreyfus, 1986)

| Стадия | Тип знания | Степень компиляции |
|--------|-----------|-------------------|
| Novice | Чистый explicit, следует правилам | 0% — читает Pack |
| Advanced Beginner | Распознаёт ситуации через опыт | ~20% |
| Competent | Выбирает план, приоритизирует | ~50% — частично скомпилировано |
| Proficient | Интуитивно схватывает ситуацию | ~80% |
| Expert | «Видит» правильное, не рефлексирует | ~100% — полностью скомпилировано |

Ключевой вывод: переход Competent → Expert — **качественная трансформация**, а не «больше знаний».

## 7. Компиляция у LLM vs у человека

| Аспект | Человек | LLM |
|--------|---------|-----|
| Tacit knowledge | 3 формы (кодифицируемое + нюансы + **воплощённое**) | 2 формы (без воплощённого) — Lu, 2025 |
| Сборка знаний | Few-shot реорганизация (Nelli et al., 2023) | Не может без специальных модификаций |
| Метакогниция | Асимметричная (Steyvers & Peters, 2025) | Ограниченная, симметричная |
| Компиляция | Declarative → procedural (перманентно) | Параметрическое знание (фиксировано после обучения) |
| Де-компиляция | Частичная (Поланьи: больше знаем, чем можем сказать) | LLM начинают помогать в извлечении tacit knowledge (Industry 5.0, 2024) |

## 8. Ключевое наблюдение

Без письма сигналы остаются слабыми и подвергаются прунингу. Мышление письмом — это не «запись мыслей», а активное строительство нейромаршрутов.

## 9. Следствия для платформы

| Компонент | Роль в модели компиляции |
|-----------|-------------------------|
| **Pack** | Исходный код (отчуждённое знание) |
| **Бот (Feed)** | Компилятор — подаёт фрагменты Pack в нужном порядке с привязкой к контексту |
| **Digital Twin** | Отражает состояние компиляции (что скомпилировано, что нет) |
| **Evaluator (I8)** | Проверяет качество компиляции (Bloom-aware: Remember? Apply? Create?) |
| **Knowledge Extractor** | Де-компилятор — извлекает из мировоззрения эксперта отчуждаемое знание |
