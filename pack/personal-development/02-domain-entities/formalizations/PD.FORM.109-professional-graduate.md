# Формализация: Каталог характеристик выпускника-Мастера

## 1. Назначение

**Проблема:** программа РР объявляет на выходе квалификацию «Мастер» (ШСМ ст. 6). Каждое наставническое решение требует ответа на вопрос «выпустить ли этого человека из РР как Мастера?». Без операционного gate ответ субъективен.

**Решение:** каталог из 8 характеристик с целевыми baseline и двойным gate.

**Три функции каталога:**

1. **Норматив выпускника.** Для каждой характеристики — целевой baseline-индекс на ст. Мастер
2. **Маршрутизация воздействия Портного.** Просадка характеристики → выбор воздействия из 6 руководств РР
3. **Архитектура измерений.** Источники данных (события, артефакты, разборы)

**Различение со FORM.093 (для ЛР):**
- FORM.093 — характеристики Проактивного ученика (ЛР), вход в РР
- FORM.109 — характеристики Мастера (выход РР), вход в ИР или продолжение в Профессионале

---

## 2. Принцип каталога

Все 8 характеристик **наблюдаемы извне** (Object ≠ Description, FPF A.7). Не путать с рычагами (FORM.108 RCS — рычаги; FORM.109 — выходные характеристики).

| Характеристика | Категория | Целевой baseline ст. Мастер |
|---|---|---|
| 1. **Шедевр предъявлен** | gate (binary) | да |
| 2. **Прикладное мастерство в домене** | gate | ≥4 |
| 3. **Команда работает по инженерным методам** | gate | ≥4 |
| 4. **Культура работы изменена** | gate | ≥4 |
| 5. **Наставничество начинающих** | gate | ≥1 ученик-Интеллектуал прошёл свою защиту |
| 6. **Системное мышление** | supporting | ≥4 |
| 7. **Калибр компании** | supporting | ≥4 |
| 8. **Регулярность практики** | supporting | ≥3 |

**Категоризация:**
- **gate (5 характеристик: 1-5)** — обязательны для квалификации Мастер
- **supporting (3 характеристики: 6-8)** — желательны; просадка → план поддержки, не отказ

## 3. Карточки характеристик

### 3.1. Шедевр предъявлен (gate, binary)

**Определение.** Артефакт выдающегося качества (MIM.WP.002), демонстрирующий мастерство в реальной задаче. Не «прошёл резидентуру», а **предъявил конкретный артефакт**, который был принят коллегиально.

**Что наблюдается:**
- Артефакт физически существует и может быть передан/показан
- Артефакт принят коллегиальным разбором (методсовет МИМ)
- Артефакт является **продуктом** (поставляется в надсистему), не только рабочим продуктом (D.160)

**Источник данных:** MIM.WP.002 как зарегистрированный артефакт + протокол разбора методсоветом.

### 3.2. Прикладное мастерство в домене (gate, ≥4)

**Определение.** Способность производить рабочие продукты в конкретной предметной области с предсказуемым качеством и скоростью.

| Idx | Признак |
|---|---|
| 1 | Domain выбран, методы не освоены |
| 2 | Применяет методы в типовых задачах с подсказкой |
| 3 | Применяет SoTA в знакомых задачах; Pack-черновик ≥10 понятий |
| 4 | SoTA освоен; ≥3 РП/мес устойчиво; Pack peer-reviewed |
| 5 | Создаёт новые методы в домене; влияет на SoTA |

**Источник данных:** темп закрытия РП (WP-context), Pack-метрики (peer-review статус), git-активность по проектам.

### 3.3. Команда работает по инженерным методам (gate, ≥4)

**Определение.** Команда до 10+ человек применяет инженерные методы (рациональная работа, системное мышление, разделение деятельностных ролей) в повседневной работе.

| Idx | Признак |
|---|---|
| 1 | Команда формально существует; методы не применяются |
| 2 | Команда применяет 1-2 метода эпизодически |
| 3 | Команда применяет ≥3 метода регулярно; есть видимые ритуалы |
| 4 | Команда работает по инженерному процессу; видимый рост производительности |
| 5 | Команда — пример для других в компании; сама становится наставниками |

**Источник данных:** интервью с членами команды + анализ командных артефактов (планы, ретро, ритуалы).

### 3.4. Культура работы изменена (gate, ≥4)

**Определение.** Видимое изменение собственной культуры работы по сравнению с состоянием до программы РР.

| Idx | Признак |
|---|---|
| 1 | Нет видимых изменений |
| 2 | Есть 1-2 новые привычки, неустойчивые |
| 3 | Видимые ритуалы и артефакты устойчивы ≥3 мес |
| 4 | Культура изменена системно: ритм + методы + артефакты + наставничество |
| 5 | Изменения сохраняются в стрессовых условиях; передаются другим |

**Источник данных:** документация «было / стало», свидетельства близкого круга (D.163 культура — внешне наблюдаема).

### 3.5. Наставничество начинающих (gate, бинарный)

**Определение.** ≥1 ученик-Интеллектуал прошёл собственную защиту (на ст. Специалист или Стратег ШСМ) под наставничеством Мастера.

**Источник данных:** записи разборов; протоколы защит подопечных.

### 3.6. Системное мышление (supporting, ≥4)

**Определение.** M4 рычаг (FORM.094) на уровне «навык» — системный разбор применяется автоматически.

**Источник:** RCS Интеллектуала (FORM.108 cp.sys + cp.int).

### 3.7. Калибр компании (supporting, ≥4)

**Определение.** Способность действовать на масштабе компании (~100-1000 человек), а не только команды.

| Idx | Признак |
|---|---|
| 1 | Действует только в своей команде |
| 2 | Инициирует кросс-командные проекты эпизодически |
| 3 | Регулярно влияет на смежные команды |
| 4 | Инициирует проекты организационного развития в масштабе компании |
| 5 | Признан экспертом организационного развития на уровне компании |

**Источник данных:** журнал инициированных проектов + их масштаб.

### 3.8. Регулярность практики (supporting, ≥3)

**Определение.** Удержание ритма работы по освоенным методам после получения квалификации. Связано с MIM.R.004: «не практикуешь — не мастер».

**Источник данных:** WakaTime / Activity Hub за 12 месяцев после защиты; устойчивость артефактов производства.

**Особенность:** низкий baseline (≥3, не ≥4) — на ст. Мастер регулярность обеспечивается командой и культурой; индивидуальная регулярность важна, но не блокирующая.

## 4. Двойной gate выпуска РР (Мастер)

```
metric_gate_master = (
    masterpiece_present == True AND
    applied_mastery_idx >= 4 AND
    team_method_adoption_idx >= 4 AND
    culture_change_idx >= 4 AND
    intellectual_student_count >= 1 AND
    systems_thinking_idx >= 4 AND       # supporting но желательно
    company_scale_idx >= 4 AND          # supporting
    practice_regularity_idx >= 3         # supporting
)

formal_defense_gate = (
    residency_completed == True AND      # MIM.FMT.003
    methodological_council_decision == "approved"
)

master_ready = metric_gate_master AND formal_defense_gate
```

**Срок действия квалификации:** 3 года. Подтверждение через повторный разбор. Не практикуешь — не мастер (MIM.R.004).

## 5. Маршрутизация воздействия (для Портного)

| Просевшая характеристика | Основной материал воздействия |
|---|---|
| Прикладное мастерство | Углубление SoTA домена; Pack peer-review |
| Команда по методам | Руководства 5+6 РР (Инженерия личности + Системный менеджмент); коучинг команды |
| Культура работы | Заземление методов в стиле жизни (METHOD.034); эпистемический ритм |
| Наставничество | Поиск ученика-Интеллектуала; регулярные разборы |
| Системное мышление | Возврат к Руководству 2 (Системное мышление); RCS Интеллектуала FORM.108 |
| Калибр компании | Кросс-командные проекты; внешний наставник на уровне компании |
| Регулярность | Поддержание гигиенического минимума ЛР; ритм инвестированного времени |

## 7. Открытые вопросы

1. **Калибровка строгости gate.** Все 5 gate + 3 supporting — это, грубо, P(прохождение) ≈ 5% за стандартный цикл РР. Если это слишком строго (большинство застревает на ст. Практик) — ослабить supporting до ≥2.

2. **Подтверждение квалификации каждые 3 года.** Что измеряется при подтверждении? Те же характеристики или их подмножество? Нужна отдельная формализация процедуры подтверждения.

3. **Граница «компания»** в характеристике 3.7. Для маленьких компаний (10-30 человек) калибр «компания» = калибр «команда» по объёму. Нужна нормализация по размеру организации.

4. **Связь шедевра с домена.** Что считается шедевром в кросс-доменной деятельности (например, организационное развитие)? Источник MIM.WP.002 даёт общее определение, но домен-специфика не описана.
