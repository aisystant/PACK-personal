# Формализация: Категории заметок (Note Categories)

## 1. Определение

**Категория заметки** — тип мысли, зафиксированной в fleeting-notes, определяющий маршрут её обработки. Категоризация выполняется при Note-Review (не при захвате — захват должен быть без трения).

Ключевой принцип: **одна заметка → одна категория → один маршрут**. Если заметка содержит несколько типов — разбить на части.

## 2. Семь категорий

| # | Категория | Суть | Маршрут-назначение |
|---|-----------|------|--------------------|
| 1 | **НЭП** | Дискомфорт, желание изменить что-то в жизни или системе. Эмоциональный сигнал | Dissatisfactions.md |
| 2 | **Задача** | Конкретное действие: «сделать X», «изучить Y», «обдумать гипотезу Z» | WeekPlan / DayPlan |
| 3 | **Знание доменное** | Паттерн, правило, различение — уже понятое, применимо вне реализации | captures.md → Pack |
| 4 | **Знание реализационное** | Как устроено в конкретной системе, архитектурный выбор | DS docs/ (или Задача: задокументировать) |
| 5 | **Черновик** | Зерно поста/статьи с концептами. Привязывается к существующему черновику в drafts/ | drafts/ (после согласования с drafts/index.md) |
| 6 | **Личные данные** | Контакт, аккаунт, токен, личная информация | personal/ |
| 7 | **Шум** | Дубль, тест, случайное, уже реализовано в РП, нерелевантное | Архив (удалить) |

## 3. Поглощения (что убрано)

| Убрана категория | Поглощена |
|-----------------|-----------|
| Бот-замечание | → Задача (тип 2) |
| Гипотеза | → Задача («обдумать гипотезу»); после обдумывания → Strategy.md вручную |
| Идея 🔄 | → Задача или Шум |

## 4. Правила маршрутизации

- **НЭП** не становится Задачей автоматически — сначала фиксируется эмоция, потом человек решает: создать РП или оставить в Dissatisfactions.md
- **Черновик**: Note-Review читает `drafts/index.md` (одна строка = название + 5 слов о чём) и предлагает подходящий существующий черновик. Новый черновик создаётся только если нет подходящего.
- **Знание доменное** требует captures.md → затем Capture-to-Pack по правилу IWE
- **Шум** — удалить без сожаления. Антипаттерн: хранить «на всякий случай»

## 5. Связь с Note-Review

Note-Review — агент, который выполняет классификацию по этим категориям и строит таблицу предложений в DayPlan (секция «Разбор заметок»). Агент предлагает — человек принимает решения.

```
fleeting-notes.md
  ↓ Note-Review классифицирует по 7 категориям
  ↓
DayPlan §Разбор заметок — таблица с предложениями
  ↓ Пользователь: ✅ / меняет маршрут / оставляет
  ↓
Целевые документы
```
