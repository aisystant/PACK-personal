# Формализация: Заземление

## 1. Тезис

**Заземление — это соединение абстрактного понятия с конкретной ситуацией из физического мира.** Понятие, не заземлённое в материальный объект или наблюдаемое действие, существует только как словесная конструкция: его можно процитировать, но нельзя применить, проверить или передать другому.

Заземление — не метафора. Это операциональная норма мышления, аналогичная электрическому заземлению: контур работает только когда есть путь к «земле» — материальному уровню. Без заземления модель «парит» — даёт интуитивно правильные ответы на типовые вопросы и ломается на новой ситуации.

## 2. Структура заземления

Заземлённое понятие имеет три точки опоры:

| Опора | Что фиксирует | Пример для понятия «ритм» |
|-------|---------------|---------------------------|
| **Материальный объект** | На каком физическом носителе понятие проявляется | Календарь с защищёнными слотами, журнал ритма |
| **Наблюдаемое действие** | Какие конкретные действия делает носитель понятия | «В 9:00 каждый будний день — слот 1.5ч на саморазвитие» |
| **Проверяемый признак** | По чему сторонний наблюдатель определит, есть понятие или нет | За 4 недели: ≥16 слотов выполнено, разница «план vs факт» ≤20% |

Если хотя бы одна опора отсутствует — понятие не заземлено, остаётся в режиме декларации.

**Пример заземления понятия «прогулка»**: абстрактное «прогулка» не имеет опор и существует только словесно. Заземление: материальный объект — конкретный человек (идущий от дома к озеру); наблюдаемое действие — перемещение неторопливым шагом (метод, отличающий прогулку от пробежки); проверяемый признак — изменение положения объекта в пространстве за интервал времени. «Скорость» при этом заземляется не как объект, а как характеристика того же материального объекта во время движения (одного порядка с температурой).

## 3. Уровни заземления (диагностика)

| Уровень | Признак | Состояние освоения |
|---------|---------|--------------------|
| 0 — Цитирование | Может процитировать определение | Знает термин |
| 1 — Применение к чужому примеру | Распознаёт понятие в чужом кейсе | Знает применение |
| 2 — Применение к своей ситуации | Находит понятие в собственной работе | Заземление слабое |
| 3 — Изменение через понятие | Использует понятие, чтобы что-то изменить в своей деятельности | Заземление прочное |
| 4 — Передача через понятие | Может объяснить и научить другого через материальный пример | Заземление полное |

Цель программы ЛР для каждого ключевого понятия — уровень 3+, не уровень 0-1.

## 4. Связь с моделированием

Заземление — это **тест модели об физический мир** (см. PD.FORM.102 «Мышление как моделирование»). Модель проверяется не цитированием, а попыткой применить к незнакомой ситуации. Если модель применима — заземление есть; если применение разваливается — модель не достроена в части заземления (опор), даже если её определения и связи выглядят правильно.

Различение D.159 (материальный объект ≠ описание) даёт критерий: заземление — это всегда движение от уровня описания (понятия, модели) к уровню материального объекта (конкретной ситуации, наблюдаемого действия). Обратное движение (от объекта к описанию) — концептуализация. Цикл «концептуализация → заземление → концептуализация» = здоровое освоение; обрыв на «концептуализация» = режим отказа PD.FAIL.070.

## 5. Заземление в программе ЛР

| Канал | Где работает заземление |
|-------|-------------------------|
| Освоение понятия | После каждого нового понятия — упражнение «найди этому 1 пример из своей работы за последнюю неделю» |
| Диагностика ступени (R28) | Прокси-вопросы измеряют не знание определений, а заземлённость: есть ли защищённый слот, есть ли рабочие продукты |
| Навигация (R27) | На «зачем мне это» — ответ через конкретную ситуацию из контекста пилота, не через абстрактную пользу |
| Оценка (Аттестатор) | Артефакты-доказательства — это материальные носители заземления |

## 6. Связи

- **PD.FORM.097 §1** — мировоззрение = главная модель. Заземление — операция проверки этой модели об физическую реальность. Без заземления мировоззрение остаётся декларацией, не исполняемым кодом.
- **PD.FORM.102** — мышление = моделирование. Заземление = одна из двух операций над моделями (вторая — концептуализация = подъём от частного к общему).
- **PD.FAIL.070** — концептуализация без заземления. Без формализации заземления режим отказа не диагностируется.
- **D.159** — материальный объект ≠ описание. Заземление = направленное движение от описания к объекту.
- **PD.METHOD.034** — практика заземления как метод.
