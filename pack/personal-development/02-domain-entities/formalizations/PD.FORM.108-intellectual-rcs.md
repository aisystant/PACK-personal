# Формализация: RCS Интеллектуала

## 1. Назначение

**Role Characterisation Space (RCS) Интеллектуала** — набор диагностических срезов, которыми измеряется зрелость мыслительного мастерства на ступенях программы Рабочего развития (PD.FORM.106): Работник → Стратег → Специалист.

**Что описывает документ:**
- 6 новых cp-срезов специфичных для Интеллектуала (cp.cnc, cp.mod, cp.sys, cp.trn, cp.dom, cp.met)
- 4 переиспользованных среза из FORM.089 (cp.wld, cp.int, cp.knw, cp.agt) — продолжают расти на РР
- Шкала 1-5 для каждого среза с операционными признаками
- Двойной gate для перехода Интеллектуал → Профессионал (выпуск из «фазы Интеллектуала» программы РР)

**Что НЕ описывает:**
- Характеристики выпускника Мастера (это PD.FORM.109 в разработке)
- Bh-характеристики (поведенческие метрики) — переиспользуются из FORM.089
- Алгоритм Диагноста — описан в PACK-MIM

**Различение со среза FORM.089:**
FORM.089 описывает RCS Ученика — измерение зрелости М1 (собранность), М2 (IWE/ОРЗ), частично М4 (системное мышление). FORM.108 фокусируется на М4 (мыслительное мастерство) и расширяет его для измерения на ступенях РР.

## 2. Структура RCS Интеллектуала

### 2.1. Срезы по группам

| Группа | Срез | Что измеряет | Расположение в FORM.094 |
|---|---|---|---|
| **Понятийная работа** | cp.cnc | Понятие vs запоминание определения | M4 |
| **Моделирование** | cp.mod | Моделирование vs воспроизведение цитаты | M4 |
| **Системный разбор** | cp.sys | Применение различений на новых задачах | M4 |
| **Перенос теории в практику** | cp.trn | Применение принципа в новом домене | M4 + W |
| **Формирование домена** | cp.dom | Есть ли направление SoTA, в котором учатся | бридж к M3 |
| **Выбор метода** | cp.met | Адекватность метода ситуации | M4 |
| **Зрелость мировоззрения** | cp.wld (из FORM.089) | Мировоззренческая дуга | W |
| **Системное мышление** | cp.int (из FORM.089) | M4 рычаг | M4 |
| **Предметные знания** | cp.knw (из FORM.089) | M3 рычаг (формирующийся) | M3 |
| **Агентность** | cp.agt (из FORM.089) | Самоинициация в проектах | A |

**Различение D.151 (Slice ≠ Slot):** cp.* — это **срез** = диагностический инструмент на момент. Срез не статичен — каждый замер даёт новое значение. Не путать со «слотом» (постоянная характеристика в структуре).

### 2.2. Сводная шкала уровней (1-5)

| Уровень | Что значит |
|---|---|
| 1 | Не различает / не применяет / не наблюдает |
| 2 | Различает в теории; применяет с подсказкой |
| 3 | Применяет систематически в знакомых ситуациях |
| 4 | Применяет автоматически в новых ситуациях; передаёт другим |
| 5 | Создаёт новые приёмы; обосновывает границы применимости |

## 3. Карточки срезов

### 3.1. cp.cnc — Понятийная работа

**Что измеряет:** способность работать с понятием (узлом в графе со связями), а не с термином (словом с определением).

| Idx | Признак baseline |
|---|---|
| 1 | Воспроизводит определение по запросу; не различает понятие и термин |
| 2 | Может объяснить понятие на знакомых примерах; не строит связи |
| 3 | Строит связи понятия с другими (≥3 связи); использует понятие в собственной речи |
| 4 | Применяет вопрос-якорь «описание чего?» (PD.METHOD.035) к спорным утверждениям; различает разные слои онтологии (объект / описание / роль) |
| 5 | Создаёт новые понятия по необходимости; обосновывает их через FPF (Object/Description/Carrier) |

**Источник:** PD.FORM.101 (заземление), PD.METHOD.035 (вопрос-якорь), PD.FORM.102 (моделирование).

**Тип данных:** auto + self-report.
**Auto-композит:** анализ собственных рабочих заметок (NLP-классификатор на определения vs связи).
**Self-композит:** Диагност-разбор: задание построить модель нового понятия на новом материале.

### 3.2. cp.mod — Моделирование

**Что измеряет:** способность строить модель ситуации вместо воспроизведения формулировки.

| Idx | Признак baseline |
|---|---|
| 1 | Цитирует источник; на вопрос «как ты это видишь?» воспроизводит чужую формулировку |
| 2 | Может перерисовать чужую схему; не перестраивает при изменении ситуации |
| 3 | Строит схему: что есть объект, какие у него части и связи; модель воспроизводится |
| 4 | Перестраивает модель при изменении ситуации; различает зависимость модели от строителя (одна ситуация → разные модели у разных профессионалов) |
| 5 | Создаёт новые модели; объясняет их через приёмы моделирования FPF |

**Источник:** PD.FORM.102 (мышление как моделирование), D.158 (моделирование ≠ запоминание).

**Тест переноса:** применить понятие к незнакомой ситуации. Не справился — cp.mod = 2; справился частично — 3; полно — 4; создаёт новую модель — 5.

### 3.3. cp.sys — Системный разбор

**Что измеряет:** применение системных различений (роль/исполнитель, описание/воплощение, метод/инструмент, целевая система/системы создания) в реальных задачах.

| Idx | Признак baseline |
|---|---|
| 1 | Не различает систему и процесс; не применяет различения |
| 2 | Различает 2-3 базовых различения (D.139, D.140, D.141) в знакомых задачах |
| 3 | Применяет 5-7 различений в новых контекстах; строит системные модели задач |
| 4 | Применяет различения автоматически; распознаёт целевую систему, надсистему, системы создания на новых проектах |
| 5 | Создаёт новые различения по необходимости; обосновывает через FPF (Strict Distinction A.7) |

**Источник:** PD.FORM.080 §3.1 (M4 системное мышление), PD.FORM.099 #1 (ясность).

**Тип данных:** Диагност-разбор + анализ WP-контекстов (есть ли структурный язык).

### 3.4. cp.trn — Перенос теории в практику

**Что измеряет:** способность применить принцип из одного домена к новому домену.

| Idx | Признак baseline |
|---|---|
| 1 | Не различает «знание» и «применение»; «прочитал — значит знаю» |
| 2 | Применяет принцип в том же домене, где его освоил |
| 3 | Применяет в смежном домене с подсказкой |
| 4 | Применяет в принципиально новом домене самостоятельно; ≥2 случая переноса задокументировано |
| 5 | Регулярно делает trans-disciplinary переносы; формулирует обобщения, применимые в нескольких доменах |

**Источник:** FPF-B (Transdisciplinary), PD.FORM.099 #4 (чувство нестыковки — катализатор переноса).

**Тип данных:** Диагност-разбор (просьба применить освоенное к незнакомой задаче) + анализ WP-портфеля.

### 3.5. cp.dom — Формирование домена

**Что измеряет:** есть ли у человека сформированная предметная область, в которой он хочет владеть SoTA.

| Idx | Признак baseline |
|---|---|
| 1 | Нет домена; интересы разрозненны |
| 2 | Заявляет несколько доменов; не различает уровень погружения |
| 3 | Назвал 1 домен; начал формировать собственный понятийный аппарат (Pack-черновик) |
| 4 | Pack-черновик ≥10 связанных понятий; домен документирован в личной стратегии |
| 5 | SoTA в домене освоен; Pack peer-reviewed; домен опубликован |

**Источник:** PD.FORM.091 §3 (ячейка «структура × мировоззрение» = деятельностный кругозор), PD.FORM.096 (выбор области интересов).

**Особенность:** cp.dom — бридж между Интеллектуалом и Профессионалом. На ст. Стратег→Специалист (РР) этот срез растёт от 1 до 3-4. На ст. Практик активизируется cp.knw (предметные знания).

### 3.6. cp.met — Выбор метода

**Что измеряет:** способность выбирать метод адекватно ситуации, обосновывать выбор, фиксировать границы применимости.

| Idx | Признак baseline |
|---|---|
| 1 | Использует «любимый» метод; не объясняет выбор |
| 2 | Следует руководствам как стандартам деятельности (compliance) |
| 3 | Выбирает корректный метод по ситуации; объясняет почему |
| 4 | Объясняет границы применимости метода; различает где метод работает / не работает |
| 5 | Предлагает улучшения в руководства там, где они плохо срабатывают (Реформатор-уровень) |

**Источник:** MIM.M.004 шкала «Методологическая дисциплина» (Ad hoc → Compliance → SoTA → Реформатор) — прямое соответствие.

**Связь с MIM.M.004:** этот срез — RCS-проекция шкалы «Методологическая дисциплина» MIM-рамки. Замер cp.met = замер дисциплины в MIM-смысле.

### 3.7. Переиспользованные срезы из FORM.089

| Срез | Что измеряет | Динамика на РР |
|---|---|---|
| **cp.wld** | Зрелость мировоззрения по дуге (PD.FORM.087 §5) | На ст. Работник: 3; Стратег: 3-4; Специалист: 4; Практик: 4-5; Мастер: 5 |
| **cp.int** | M4 рычаг (системное мышление) | Растёт от 2 (Работник) до 4-5 (Специалист+) |
| **cp.knw** | M3 рычаг (предметные знания) | Появляется на ст. Специалист; растёт до зрелости на ст. Мастер |
| **cp.agt** | Агентность (самоинициация) | Должен быть ≥4 на входе в РР (выпуск ЛР); растёт до 5 на ст. Мастер |

> **Связь с FORM.089:** RCS Интеллектуала — расширение, не замена. На ст. Работник cp.wld/cp.int/cp.agt продолжают замеряться по правилам FORM.089. С ст. Стратег добавляются срезы FORM.108 (cp.cnc, cp.mod, cp.sys, cp.trn, cp.dom, cp.met).

## 4. Целевые baseline по ступеням РР

> Источник: PD.FORM.106 (фокусы ступеней) + PD.FORM.107 (нормативная матрица).

### 4.1. Сводная таблица целевых индексов

| Срез | Работник | Стратег | Специалист | Практик | Мастер |
|---|---|---|---|---|---|
| cp.wld (мировоззрение) | 3 | 3-4 | 4 | 4-5 | 5 |
| cp.int (M4) | 2 | **3-4** | **4** | 4 | 4-5 |
| cp.cnc (понятия) | 2 | 3 | **4** | 4 | 4-5 |
| cp.mod (моделирование) | 2 | **3** | **4** | 4 | 4-5 |
| cp.sys (системный разбор) | 2 | **3-4** | **4** | 4 | 4-5 |
| cp.trn (перенос) | 1-2 | 2-3 | **4** | 4 | 4-5 |
| cp.dom (домен) | 1 | 2 | **3-4** | 4-5 | 5 |
| cp.met (выбор метода) | 2 | **3** | **3-4** | 4 | 4-5 |
| cp.knw (предметные знания) | 1 | 1-2 | 2-3 | **3-4** | 4-5 |
| cp.agt (агентность) | 4 | 4 | 4 | 4-5 | 5 |

**Жирным выделены ведущие срезы для каждой ступени** (где Портной делает основной фокус воздействия).

### 4.2. Двойной gate выпуска из «фазы Интеллектуала» программы РР (Специалист → Практик)

```
metric_gate_intellectual = (
    cp.wld >= 4 AND
    cp.int >= 4 AND
    cp.cnc >= 4 AND
    cp.mod >= 4 AND
    cp.sys >= 4 AND
    cp.trn >= 4 AND
    cp.dom >= 3 AND
    cp.met >= 3 AND
    cp.agt >= 4
)

# Условие 2: «инженерный процесс предъявлен на собственном проекте»
real_application_gate = (
    pack_draft >= 10 связанных понятий AND
    transfer_cases >= 2 (документированы) AND
    team_role_split >= 1 проект (с явным разделением)
)

graduate_intellectual_ready = metric_gate_intellectual AND real_application_gate
```

### 4.3. Двойной gate выпуска программы РР (Практик → Мастер)

> См. PD.FORM.107 §8. Здесь приводится связь с RCS:

```
metric_gate_master = (
    все cp.* >= 4 кроме cp.knw >= 4-5 AND
    cp.dom >= 4-5 AND
    cp.knw >= 4-5  # Профессионал-зрелость
)

# Условие 2: шедевр + резидентура
mastery_gate = (
    masterpiece (MIM.WP.002) предъявлен AND
    residency (MIM.FMT.003) пройдена AND
    team_culture_change задокументирована AND
    intellectual_student >= 1 (свой ученик прошёл защиту)
)

master_ready = metric_gate_master AND mastery_gate
```

## 5. Анти-Goodhart защита: связь с внутренними маркерами (PD.FORM.099)

> Цель: предотвратить «оптимизацию под опросник» — когда человек даёт «правильные» ответы без реального мыслительного сдвига.

| Cp-срез RCS | Внутренний маркер из PD.FORM.099 |
|---|---|
| cp.cnc | #1 Ясность (видишь устройство ситуации) |
| cp.mod | #5 Лёгкость переключения уровней абстракции |
| cp.sys | #1 Ясность + #4 Чувство нестыковки |
| cp.trn | #4 Чувство нестыковки (катализатор переноса) |
| cp.dom | #7 Любопытство как фон |
| cp.met | #3 Калиброванная уверенность (в способе работы, не в результате) |
| cp.wld | #6 Чувство масштаба собственного незнания (counter-balance) |

**Правило:** на gate ≥4 для cp.* должны присутствовать соответствующие маркеры на уровне F или G (см. FORM.099 §2.2). Если cp.cnc = 4, но маркер #1 (ясность) на уровне R (иллюзорный) — это сигнал «оптимизации без сдвига». Диагност понижает оценку.

## 6. Алгоритм Диагноста при работе с RCS Интеллектуала

> Расширение алгоритма из FORM.089 §6.

### 6.1. Три фазы диагностики

1. **Первичный срез (3-4 вопроса):** якорные вопросы по cp.sys, cp.mod, cp.met, cp.dom — самые информативные для определения ступени РР. Применяется CAT-принцип (начало с предполагаемой ступени, не с минимума).

2. **Уточнение bottleneck (1-2 вопроса):** drill-down по слабому срезу (idx < 3). Если все ≥ 3 — пропускается.

3. **Аудит маркеров (0 вопросов, автоматически):** анализ заметок и WP-контекстов на присутствие маркеров FORM.099. Если cp.* = 4-5, а маркеры на уровне R — понижение.

### 6.2. Вычисление ступени РР по RCS

```python
stage_intellectual = min(cp.cnc, cp.mod, cp.sys, cp.trn, cp.met)
stage_creative = min(cp.dom, cp.knw)
bottleneck_slot = argmin(all)
rp_stage = "Работник" if min(W,M4) < 3 else "Стратег" if cp.sys < 4 else "Специалист" if cp.trn < 4 else "Практик" if cp.knw < 4 else "Мастер"
```

### 6.3. Макс. 5 вопросов

После 5 вопросов — определять на основе имеющегося. Не зачитывать все 10 срезов в диалоге — это утомляет и снижает качество ответов.

## 9. Открытые вопросы (калибровка)
