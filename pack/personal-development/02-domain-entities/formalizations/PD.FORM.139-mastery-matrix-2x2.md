# Формализация: Матрица мастерства 2×2 (человек + ИИ-инструмент)

## 1. Определение

**Матрица мастерства 2×2** — частный случай матрицы стиля жизни (PD.FORM.060), специализированный для системы «человек + ИИ-инструмент». Разделяет виды мастерства созидателя по двум осям:

- **Субъект** — на что направлено мастерство: на **себя** (человек) или на **инструмент** (ИИ-ассистент, экзокортекс).
- **Режим** — тип работы: **управление** (применение уже освоенного, удержание нормы) или **развитие** (освоение или создание нового, поднятие нормы).

Отличие от родительской матрицы PD.FORM.060: там инструмент входит в общую область «Среда». В системе с ИИ инструмент становится половиной рабочей системы «человек + ИИ», поэтому здесь он выделен в **отдельную ось субъекта** наравне с «Я».

## 2. Структура (матрица 2×2)

Каждая ячейка — **класс методов**: набор методов соответствующего вида мастерства.

| Субъект | Управление (удержание нормы) | Развитие (поднятие нормы) |
|---|---|---|
| **Я** (человек) | **Методы управления собой** — внимание, состояние, собранность, удержание слота | **Методы развития себя** — освоение новых методов саморазвития, компиляция навыков |
| **Инструмент** (ИИ) | **Методы владения инструментом** — работа оператора с готовыми функциями, скиллами, командами | **Методы развития инструмента** — создание своих скиллов, скриптов, баз знаний, процессов |

## 3. Связь с матрицей стиля жизни (PD.FORM.060)

| Ось этой матрицы | Ось PD.FORM.060 |
|---|---|
| Режим: Управление / Развитие | = режим Поддерживать / Развивать |
| Субъект: Я | = область «Я» |
| Субъект: Инструмент | = выделенная под-область «Среда» (инструменты) |

Матрица 2×2 — приближение (zoom-in) родительской матрицы на строку «Я» и инструментальную часть строки «Среда», с инструментом, поднятым до отдельной оси. Конфликта нет: разные уровни детализации одного различения «поддержание ≠ развитие».

## 4. Тесты-признаки: разграничение ячеек

| Вопрос | Ячейка |
|---|---|
| После действия **я** умею что-то новое? | Да → развитие себя; нет → управление собой |
| После действия **инструмент** умеет что-то новое? | Да → развитие инструмента; нет → владение инструментом |
| Объект внимания — человек или инструмент? | человек → верхняя строка; инструмент → нижняя |

**Граница «владение ≠ развитие инструмента»:** применение готовой функции инструмента — владение; появление у инструмента новой функции (скилл, скрипт, плагин) — развитие.

## 5. Структура зависимости

Методы управления собой (верхняя-левая ячейка) — базис: без управления вниманием и состоянием методы остальных трёх ячеек нестабильны. Это та же зависимость, что «приоритет поддержания» в PD.FORM.060 (развитие без поддержания = разрушение), спроецированная на систему «человек + ИИ».

## 6. Связи

| Связь | Соотношение |
|---|---|
| PD.FORM.060 (матрица стиля жизни) | родитель; эта матрица — частный случай |
| «Удержание нормы ≠ Поднятие нормы» (distinctions.md) | ось Управление / Развитие = это различение |
| PD.D.142 (Саморазвитие ≠ Работа) | развитие себя + развитие инструмента = саморазвитие; управление использует их плоды |
| DP.D.046 (Экзоскелет ≠ Автопилот) | владение инструментом = осознанное усиление, не делегирование мышления |
| «Пользователь ИИ ≠ Разработчик агентов» (distinctions.md) | переход «владение инструментом» → «развитие инструмента» = качественный сдвиг калибра |
