# Формализация: Трекер слотов

## 1. Определение

Трекер слотов -- система записи впечатлений, опытов и знаний, которые агент получает, с привязкой к пробелам в развитии. Позволяет видеть, какие «слоты» (области развития) заполняются, а какие остаются пустыми.

## 2. Различение: Трекер слотов ≠ Система публикации

| Аспект | Трекер слотов | Система публикации |
|--------|--------------|-------------------|
| Цель | Учёт для развития | Выход для аудитории |
| Что записывает | Впечатления, опыт, знания | Готовые тексты, посты, статьи |
| Для кого | Для агента (внутренний) | Для внешних потребителей |
| Метрика | Заполненность пробелов | Охват, реакции |
| Пример | «Прочитал главу X — закрыл пробел Y» | «Опубликовал пост о Y» |

Путаница: когда трекер слотов превращается в витрину («что я выложил»), он перестаёт работать как инструмент развития и становится инструментом публикации.

## 3. Структура слота

Слот = область развития, которая может быть:
- **Пустая** -- пробел не закрыт, впечатлений нет
- **Частично заполненная** -- есть активные впечатления (PD.FORM.072), но недостаточно для компетенции
- **Заполненная** -- агент может действовать в этой области
