# Формализация: Модель аналогии компьютер↔человек

## 1. Определение

Структурированная аналогия между компьютерной архитектурой и устройством человека-агента. Аналогия помогает различить загрузку (чтение/потребление) от компиляции (практика/тренировка): чтение книги = загрузка source code, но не компиляция в навык. Мировоззрение — скомпилированный код, который исполняется автоматически.

## 2. Структура

| Компьютер | Человек | Комментарий |
|-----------|---------|-------------|
| Source code | Теории, знания, описания методов | Читаемый, редактируемый, но не исполняемый напрямую |
| Compiled code | Мировоззрение | Исполняется автоматически, результат «компиляции» через практику |
| Program | Мастерство (набор навыков) | Совокупность скомпилированных подпрограмм |
| OS | Интеллект | Управляет запуском программ, распределяет ресурсы |
| CPU | Мозг / тело | Аппаратная база, выполняющая инструкции |
| Execution environment | Контекст / состояние / эмоции | Влияет на производительность, не на код |

Ключевое различение: загрузка source code (чтение) ≠ компиляция (практика). Агент, прочитавший книгу, загрузил source code. Агент, применивший знания многократно, — скомпилировал навык. Компиляция требует повторений и обратной связи.
