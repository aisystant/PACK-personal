# PD.FORM.084 — Pipeline заметок: схема ВДВ

> Вход → Движение → Выход. Диагностика узких мест по состоянию на апрель 2026.

---

## Схема ВДВ

```
ВХОДЫ                   ДВИЖЕНИЕ                        ВЫХОДЫ
──────────────────────────────────────────────────────────────────
Бот Telegram            Note-Review (23:00)
(`. + текст`)     →     - классифицирует жирные    →   Dissatisfactions.md
                        - строит таблицу                WeekPlan
Ручные записи     →     «Разбор заметок»            →   captures.md → Pack
fleeting-notes.md       в DayPlan                   →   drafts/D-XXX.md
                                                    →   personal/ (accounts, contacts)
Capture-to-Pack   →     Экстрактор (on-demand)      →   Pack (PACK-personal, PACK-*)
(из сессий)             - обрабатывает captures.md  →   archive/captures/

WP-context        →     Day-Plan (4:00)             →   DayPlan «Разбор заметок»
(inbox/WP-*.md)         - переносит неразобранное       (несделанное → следующий день)
                        - строит план дня
```

## Реальное состояние на 2026-04-01

| Узел | Размер | Темп накопления | Узкое место |
|------|--------|-----------------|-------------|
| `fleeting-notes.md` | 70 строк | ~2-5 заметки/день | Жирные без TTL — зависают |
| `captures.md` | 3601 строка | ~10-20 записей/неделю | Большая — нужна регулярная очистка в archive/ |
| `drafts/` | 23 черновика (D-001..D-023) | ~1-2/неделю | Нет автоматического маршрута «заметка → черновик» |
| `personal/` | 4 файла (accounts, contacts, reference, secrets) | Редко | Маршрутизация личных данных работает |
| `Dissatisfactions.md` | 105 строк | ~1-2/неделю | Работает, регулярно обновляется |

## Узкие места (приоритизированы)

### У1 — Жирные заметки без TTL (⛔ критично)

**Симптом:** Жирная заметка = «новая, необработанная». Но в fleeting-notes.md есть жирные из прошлых недель.

**Причина:** Note-Review обрабатывает, но иногда пропускает; Day-Plan переносит, но только явно неразобранное.

**Решение (реализовано в Ф1/Ф5):** Новая архитектура — Note-Review строит таблицу в DayPlan, пользователь ставит ✅. Жирные без ✅ → переносятся автоматически.

**Ещё нужно:** Ритуал Session-Prep (Пн, 04:00) — очистка жирных старше 7 дней (Задача → WeekPlan или Шум → архив).

### У2 — captures.md накапливается (⚠️ высокий)

**Симптом:** 3601 строка. Экстрактор добавляет, Session-Prep должен очищать — но это требует ручного подтверждения пользователя.

**Причина:** captures.md = staging area. Экстрактор пишет → пользователь одобряет → уходит в Pack. Одобрение — ручное.

**Решение:** Добавить в Session-Prep (Пн) явный шаг: «Просмотреть captures.md → подтвердить или отклонить новые записи с прошлой недели». После одобрения → архив.

### У3 — Маршрут «заметка → черновик» не автоматизирован (⚠️ средний)

**Симптом:** Есть 23 черновика, но маршрутизация «жирная заметка → предложить черновик» раньше не работала.

**Причина:** Старая Note-Review не читала `draft-list.md`. Новая архитектура (Ф5) предусматривает чтение индекса черновиков.

**Решение (запланировано в Ф5):** Note-Review читает `drafts/draft-list.md` → при классификации «Черновик» предлагает конкретный файл черновика.

**Нужно:** Создать `drafts/draft-list.md` с кратким содержанием каждого черновика (одна строка = ID + 5 слов о чём).

### У4 — Нет маршрута «обдуманная гипотеза → Strategy.md год» (⚠️ средний)

**Симптом:** Гипотеза в новой схеме = Задача «обдумать». После обдумывания — пользователь сам переносит в Strategy.md (раздел «Год»).

**Причина:** Нет структурированного места в Strategy.md для «год — желания и гипотезы».

**Решение:** Добавить секцию «Год 2026 — желания и гипотезы» в Strategy.md (WP-196 Ф1 описывает структуру, Ф5 не добавил этот раздел явно — Strategy.md имеет «2026: Масштабирование экосистемы» в archive-style блоке).

**Нужно:** Преобразовать «2026: Масштабирование экосистемы» из нарратива в структуру «желания + гипотезы если→то».

## Ритуалы очистки pipeline

| Ритуал | Периодичность | Что очищается |
|--------|---------------|---------------|
| Day-Open | Ежедневно | fleeting-notes.md → DayPlan «Разбор заметок» (перенос неразобранного) |
| Day-Close / Note-Review | Ежедневно (23:00) | Жирные заметки → таблица предложений в DayPlan |
| Strategy Session (Пн) | Еженедельно | WeekPlan → план; жирные > 7 дней → Задача или Шум |
| Session-Prep (Пн, 04:00) | Еженедельно | captures.md → одобрение → archive/; WP-context done → archive/ |
| Monthly Session | Ежемесячно | Strategy.md → обновить фокус месяца; черновики без активности > 30 дней → оценить |

---

## Что работает хорошо (не трогать)

- fleeting-notes.md → Note-Review (пайплайн базово работает)
- Dissatisfactions.md (регулярно пополняется, структурирована)
- captures.md (Экстрактор + одобрение пользователя = надёжно)
- personal/ (accounts, contacts, secrets — структура адекватна)
- WP-context files в inbox/ (живые, обновляются по сессии)
