# Формализация: Состояние системы (ролевое, продуктивное, переходы)

## 1. Определение

**Состояние системы** -- динамический снимок системы «агент в роли» в конкретный момент времени. Описывает фактическое положение, а не потенциал или характеристику.

Три аспекта состояния, релевантные для созидателя:

| Аспект | Полюс «работает» | Полюс «не работает» | Диагностический вопрос |
|--------|-----------------|-------------------|----------------------|
| **Ролевое** | Функция роли исполняется (создаётся обещанный РП) | Функция заблокирована (роль занята, нет ресурса, конфликт) | Создаётся ли обещание роли прямо сейчас? |
| **Продуктивное** | Агент в рабочем режиме S2 (осознанный выбор метода) | Агент в автоматическом режиме S1 (привычка, не метод) | Я действую по методу или по инерции? |
| **Энергетическое** | Ресурс доступен: фокус, внимание, физическое состояние | Ресурс истощён: усталость, рассеянность, стресс | Есть ли сейчас ресурс для выбранного метода? |

## 2. Снапшот состояния

**Снапшот** -- фиксация состояния в момент «сейчас» для сравнения с предыдущим моментом. Инструмент диагностики, не оценки: снапшот не говорит «плохо», он говорит «вот как сейчас».

Применение снапшота:
- Перед сессией работы: какое состояние входа? (помогает выбрать метод под состояние)
- После события: как изменилось состояние? (петля обратной связи)
- При блокере: в каком состоянии застрял? (ролевое или продуктивное?)

## 3. Переход через метод

Состояние не меняется само по себе. Метод -- операция, переводящая систему из одного состояния в другое.

Примеры переходов:
- Автомат (S1) → осознанный выбор (S2): метод остановки, вопрос «что я сейчас делаю?»
- Роль заблокирована → роль активна: устранение блокера (ресурс, договорённость, инструмент)
- Истощение → восстановление: метод досуга, смены контекста

**Граница:** метод меняет состояние, но не меняет характеристику напрямую. Характеристика (PD.D.084) -- стабильное свойство, результат многих состояний во времени.
