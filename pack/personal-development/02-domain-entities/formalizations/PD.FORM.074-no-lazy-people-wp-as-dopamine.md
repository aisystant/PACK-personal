# Формализация: Нет ленивых людей — РП как дофамин

## 1. Определение

«Лень» -- не характеристика человека, а симптом неправильной настройки системы петель обратной связи. Любой человек хочет делать рабочие продукты, если получается. И отказывается от дела, если оно идёт очень сложно без видимого результата.

## 2. Механизм

```
РП получается → видимый результат → дофамин → хочется продолжать → следующий РП
РП не получается → нет результата → фрустрация → отказ (рационален!) → ярлык «лень»
```

Выполнение РП -- как наркотик: когда получается, невозможно остановиться. Сложность без видимого результата → фрустрация → отказ. Это не лень, а рациональная реакция на отсутствие подкрепления.

## 3. Импликации

| Ситуация | Ошибочная интерпретация | Правильная интерпретация |
|----------|------------------------|--------------------------|
| Человек бросает задачу | «Ленивый» | Петля обратной связи не настроена |
| Человек прокрастинирует | «Нет дисциплины» | Задача слишком крупная, нет промежуточных результатов |
| Человек горит новым проектом | «Мотивирован» | Дофаминовая петля работает (пока задачи простые) |
| Человек бросает на середине | «Не хватило воли» | Вышел за порог сложности без промежуточного подкрепления |

## 4. Практическое правило

Если агент «ленится» -- нужно не «мотивировать», а:
1. Разбить РП на более мелкие (с видимым результатом каждые 1-2 часа)
2. Убрать препятствия на входе (PD.FORM.017 -- два ключа для действия)
3. Сделать прогресс видимым (чеклисты, метрики, перечёркнутые задачи)
