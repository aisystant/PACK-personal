# Формализация: Мультипликатор IWE

## 1. Определение

**Мультипликатор IWE** — отношение суммы бюджетных часов закрытых рабочих продуктов к сумме часов учётного времени (часы у клавиатуры) за период.

```
multiplier = Σ budget_hours(wp_closed in period) / Σ accounted_hours(period)
```

Единица измерения: безразмерное число (например, 2.0 = «один час у клавиатуры превращается в два часа запланированного бюджета»). Период по умолчанию — день.

## 2. Что мультипликатор меряет, что не меряет

| Меряет | Не меряет |
|--------|-----------|
| **AI-leverage** — во сколько раз ИИ-инструменты сжимают плановую длительность работы | Качество результата |
| Соотношение плана к факту по бюджету | Полезность того, что сделано |
| Эффективность связки «пилот + IWE-инструментарий» | Объём ценности, доставленной потребителю |

Если бюджет завышен — мультипликатор завышается без улучшения работы. Если бюджет занижен — мультипликатор недооценён даже при хорошей работе. Поэтому мультипликатор не используется в одиночку как метрика мастерства.

## 3. Числитель и знаменатель — что входит

### Числитель: `Σ budget_hours(wp_closed)`

Источник: событие `wp_completed` в `domain_event` за период (`payload.date == period`).
Поле: `payload.budget_hours` — оценка бюджета РП при открытии (не пересматривается задним числом).

Что включается:
- РП, закрытые в течение периода
- Бюджет = исходная оценка при открытии РП

Что НЕ включается:
- РП, открытые но не закрытые (work-in-progress)
- РП, пересмотренные по бюджету в середине работы — берётся первоначальная оценка
- Подзадачи без формализации как РП

### Знаменатель: `Σ accounted_hours`

Источник: событие `coding_time` в `domain_event` за период.
Поле: `payload.total_seconds / 3600` — часы у клавиатуры (WakaTime).

Что включается: **учётное время** ([D.142](#d142)) — всё время у клавиатуры, включая работу за деньги, личные проекты, саморазвитие. Без доменного фильтра.

Что НЕ включается: время вне редактора (встречи, чтение, рефлексия, разговоры — даже если по делу).

## 4. Граничные случаи

| Случай | Поведение | Обоснование |
|--------|-----------|-------------|
| `accounted_hours < 0.5h` | `multiplier = null` | Малая выборка → шум, нет значимых данных |
| Нет `wp_completed` за период | `multiplier = 0` | День без закрытий — закономерный ноль, не отсутствие данных |
| `wp_completed` есть, `coding_time` нет | `multiplier = null` | Закрытие без учёта времени — измерение невозможно |
| Период >1 дня (например, 7-day rolling) | Среднее дневных мультипликаторов, не общая дробь | Защита от перекоса крупного дня |

## 5. Связь с WakaTime и DP.FM.036

WakaTime — корректный источник для **учётного времени** (всё у клавиатуры). [DP.FM.036](../../../../../PACK-digital-platform/pack/digital-platform/05-failure-modes/DP.FM.036-wakatime-coding-time-bias.md) фиксирует, что WakaTime **некорректен** как прокси **инвестированного времени** (только саморазвитие). Разные оси: мультипликатор IWE меряет учётное, индикатор `bh.inv` (PD.FORM.089) — инвестированное.

## 6. Два источника мультипликатора (автоматический и ручной)

| Источник | Вычисление | Сильная сторона | Слабая сторона |
|----------|-----------|-----------------|----------------|
| **Автоматический** | Формула §1 из `domain_event` | Объективность, воспроизводимость | Видит только формальные РП и WakaTime; не видит работы без `wp_completed` |
| **Ручной** | Оценка пилота при закрытии дня, эмитится событием `day_close` (`payload.multiplier`) | Учитывает прерванные сессии, контекст, неформализованные работы | Субъективен, зависит от состояния пилота |

**Не должны совпадать.** Расхождение между источниками — сигнал, а не ошибка:
- ручной > автоматического → есть невидимая для автомата работа (повод формализовать)
- ручной < автоматического → пилот занижает (повод проверить, не страдает ли от перфекционизма)

## 7. Использование в планировании бюджета РП

При оценке бюджета нового РП ориентироваться на **наблюдённый мультипликатор за последние 7 дней** (rolling avg), не на целевой. Целевой мультипликатор без подкреплённого факта искажает планирование в сторону недооценки бюджета.

Норма пилота 2026 — 2.0x, не 3.0x (зафиксировано WeekPlan W20, 11 мая).

## 8. Различения

| Различение | Почему важно |
|------------|--------------|
| Мультипликатор ≠ продуктивность | Продуктивность = польза/время. Мультипликатор = бюджет/время. Бюджет может быть оторван от пользы. |
| Мультипликатор ≠ скорость | Скорость = объём работы/время без учёта плана. Мультипликатор учитывает план. |
| Учётное время ≠ инвестированное время | См. [D.142](../../01-domain-contract/01B-distinctions.md). WakaTime подходит для учётного, не для инвестированного. |
| Мультипликатор ≠ метрика мастерства | Мастерство — характеристика ([PD.FORM.093](./PD.FORM.093-graduate-characteristics-vs-levers.md)). Мультипликатор — индикатор-сырьё, входящий в характеристики, но не равный им. |
