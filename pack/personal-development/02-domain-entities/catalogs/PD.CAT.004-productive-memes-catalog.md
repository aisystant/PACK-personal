# Каталог продуктивных мемов (PD.CAT.004)
<!-- index-health: skip -->
<!-- Pack catalog: продуктивный контрагент PD.CAT.001. Длинные строки = доменное содержание ячеек, не дамп контекста. -->

> **Продуктивный мем** (D.010) — компактная формулировка, которая запускает метод мышления через раскрытые, проверяемые основания: после неё человек иначе действует или мыслит и может сам вычленить, почему (уважение к самостоятельности). Контраст: непродуктивный мем либо не запускает метода (чистый аффект), либо меняет поведение в обход рассуждения (манипуляция), либо внушает беспомощность.

**Зеркало PD.CAT.001.** Этот каталог — продуктивный контрагент [PD.CAT.001](PD.CAT.001-memes-catalog.md) «Каталог непродуктивных мемов»: PD.CAT.001 = «что ломает» (64 непродуктивных мема), PD.CAT.004 = «что чинит». Колонка «Зеркало» указывает M-ID непродуктивного мема, который данный продуктивный мем перекрывает. Источник формулировок — переработанные «продуктивные антитезисы» PD.CAT.001 (проверены как самостоятельные позитивные формулировки по D.010, а не паразиты на формулировке непродуктивного мема) + реестр находок РП-333 + руководства программы ЛР.

**Статус — seed.** ≥1 продуктивный мем на каждую из 5 областей (всего 12). Полное наполнение — отдельной порцией под РП-333 Ф-M. Производный cross-domain индекс (с аннотацией риторического приёма) — [memes-index.md в PACK-rhetoric](../../../../../PACK-rhetoric/pack/rhetoric/registry/memes-index.md).

## Carry-over (отдельной порцией под РП-333 Ф-M)

- **Углубление seed → полный канон:** PD.CAT.001 содержит 64 непродуктивных мема; их продуктивные антитезисы дают ещё ~50 кандидатов после проверки D.010.
- **Кросс-доменные мемы:** P-003 (ИИ) и P-008 (окружение) имеют ведущий дом personal + ссылку во второй дом; при наполнении вторых домов (digital-platform, ecosystem) добавить обратные ссылки.
- **Программы РР и ИР:** этот seed -- срез программы личного развития (ЛР). Мемы рабочего (РР, дом PACK-MIM) и исследовательского (ИР, дома пока нет как bounded context) развития -- отдельными порциями.
- **Не-мем идеи программы:** идеи, не проходящие D.010 как мемы, маршрутизируются через Routing Gate в обычные формализации (FORM/METHOD/D), не в этот каталог.
