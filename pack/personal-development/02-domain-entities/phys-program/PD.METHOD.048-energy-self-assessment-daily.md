# [PD.METHOD.048] Ежедневная самооценка энергии

## Описание

**Метод ежедневной самооценки энергии** — процедура субъективной оценки уровня энергии в конце дня по шкале 1-5 как интегрального сигнала работоспособности и восстановления (субслой «восстановление» физического слоя ст. 0, [PD.FORM.138 §2 №4](../02-domain-entities/phys-program/PD.FORM.138-physical-layer-stage-zero.md)).

Метод производит ежедневную точку данных в шкале 1-5; недельная динамика этих точек — baseline-сигнал для субслоя «восстановление».

Метод — minimal viable proxy для биометрии восстановления (HRV, ECG, sleep stages); более точные proxy требуют wearable-устройств и валидации (PD.FORM.138 §9 пункт 3).

---

## Forces

_(Optional, WP-448 Ф12) Какие конкурирующие давления удерживает метод._

| Force | Tension |
|-------|---------|
| Субъективность оценки ↔ потребность в сопоставимом сигнале | Шкала калибруется индивидуально пилотом (§«Constraints»: «внутренняя калибровка пилота важнее точности шкалы»), но недельное среднее используется как сравнимый baseline-сигнал (§«Метрика baseline») |
| Minimal viable proxy ↔ точность биометрии (HRV/ECG) | Метод сознательно выбирает дешёвый субъективный proxy вместо wearable-метрик (§«Описание»), выигрывая в доступности и теряя в точности сигнала о восстановлении |
| Избегание gaming ↔ желание увидеть улучшение | Метод явно запрещает «улучшать» цифры (§«Avoid gaming»), но естественное желание видеть рост показателя конкурирует именно с этим требованием честной регистрации |

| Вход | Описание | Источник |

| Состояние пилота в конце дня | Субъективное ощущение энергии и готовности к новой деятельности | Внутренняя оценка пилота |

## Выход

| Выход | Описание |
|-------|----------|
| Запись `{date, energy: 1-5}` | Точка данных в Day Close (новое поле «энергия 1-5») |
| Недельное среднее | Среднее значение за 7 дней |
| Сигнал baseline-статуса | OK / просадка для субслоя «восстановление» |

---

## Шкала 1-5

| Значение | Описание состояния (саморасшифровка пилота) |
|----------|---------------------------------------------|
| **1** | Полное истощение, желание лечь. Любая дополнительная нагрузка невозможна |
| **2** | Сильная усталость, остатки энергии только на завершение текущего. Завтра потребуется восстановление |
| **3** | Базовая усталость в норме после рабочего дня. Энергии достаточно для лёгкого слота саморазвития |
| **4** | Хорошее состояние, есть энергия на дополнительную работу. Желание продолжать |
| **5** | Высокая энергия, тяга к деятельности. Ощущение «мог бы ещё час работать» |

**Baseline saneness:** среднее за неделю ≥ 3.

---

## Метрика baseline

| Параметр | Universal baseline (PD.FORM.138) | Индивидуальная калибровка |
|----------|-----------------------------------|---------------------------|
| Среднее энергии за неделю | ≥3 из 5 | Среднее за 4 недели − 0.5 (нижняя граница) |
| Минимум 1 значения | ≥2 из 5 | Не калибруется (нижний предел) |

**Правило «baseline, не оптимум»:** baseline — это уровень восстановления, при котором носитель saneness. Не оптимум (постоянно 5/5 = подозрение на эйфорию/перевозбуждение, PD.STATE.004 «Overexertion»).

## Триггер просадки

| Условие | Действие |
|---------|----------|
| Энергия <3 два дня подряд | Сигнал просадки субслоя «восстановление» от [PD.ROLE.006](../02-domain-entities/phys-program/PD.ROLE.006-body-as-first-system.md) |
| Энергия =1 один день | Сигнал просадки (один день =1 = серьёзный сигнал, не ждать второго) |
| Просадка >7 дней | Сигнал «вне scope» — рекомендация проверки сна/питания/физического состояния, при необходимости консультация |

**Действие при просадке:** следующий день — досуг приоритетнее слота саморазвития ([PD.FORM.138 §4](../02-domain-entities/phys-program/PD.FORM.138-physical-layer-stage-zero.md), см. [PD.METHOD.006 Leisure Organization](PD.METHOD.006-leisure-organization.md)).

## Constraints и Applicability

| Constraint | Описание |
|------------|----------|
| **Subjectivity** | Метод по природе субъективен; внутренняя калибровка пилота важнее точности шкалы |
| **Latency** | Оценка в Day Close (конец дня), не в течение дня; ретроспективная фиксация состояния |
| **Avoid gaming** | Не пытаться «улучшить» цифры — это measurement, не KPI; занижение/завышение разрушает сигнал |
| **Calibration period** | Первые 2 недели — определение собственной нормы; baseline пересчитывается раз в месяц |

### Когда применим
- При входе в программу ЛР (определение стартового baseline).
- Постоянно при работе в программе (ежедневная отметка в Day Close).

### Когда не применим
- В первые 2 недели после старта программы — данных для baseline ещё нет; downshift-режим по умолчанию.
- При диагностированных аффективных расстройствах — measurement остаётся, но интерпретация и план восстановления — компетенция специалиста.

## Bias-Annotation (tentative)

Tentative — карточка в статусе draft, ещё не откалибрована практикой.

_(Optional, WP-448 Ф12) Куда систематически съезжает внимание практикующего. Не дублирует «Failure Modes» — там «что пойдёт не так», здесь «куда дрейфует внимание»._

| Bias | Direction of distortion |
|------|--------------------------|
| Вечерняя оценка переоценивается в «нормальные» дни | Несмотря на явный запрет gaming (§«Avoid gaming»), внимание смещается на попытку удержать «хорошее» число, и оценка неосознанно завышается в дни с формально приемлемой нагрузкой |
| Единичное значение «1» недооценивается на фоне правила «2 дня подряд» | Метод отдельно выделяет единичный день с энергией =1 как самостоятельный серьёзный сигнал (§«Триггер просадки»), но внимание практика по умолчанию ждёт двух дней подряд, как для прочих случаев, и недооценивает срочность единичного сигнала |
