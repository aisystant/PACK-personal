# Формализация: Физический слой ст. 0 — saneness (Pre-condition программы ЛР)

## 1. Тезис

**Физический слой** — это **ст. 0 saneness**, необходимое pre-condition входа в программу личного развития (PD.FORM.080 ст. 1 «Случайный»). Без него ритм 10ч/нед инвестированного времени (PD.FORM.097 §4 «минимальный стандарт ЛР») недостижим устойчиво: организм-носитель не выдерживает нагрузку, ритм рушится, метрики проседают, мировоззренческий сдвиг не закрепляется.

**Граница** (5 различений, держать строго):

| Что физический слой ст. 0 ЕСТЬ | Что физический слой ст. 0 НЕ ЕСТЬ |
|--------------------------------|-----------------------------------|
| Проверка операционных условий (saneness check) носителя | Методы оздоровления, лечения, спорта |
| Свойство «организм работает достаточно» | Свойство «организм идеален / в форме» |
| Pre-condition для входа в программу ЛР (ст. 1) | Первая ступень программы ЛР |
| Снаружи программы (предусловие) | Внутри программы (методы развития) |
| Описывает 4 субслоя на baseline | Описывает прогресс по саморазвитию |

**Антипод** (PD.FAIL.021 «Ignoring Sleep and Daily Routine»): переход к продвинутым практикам саморазвития при сломанном сне и отсутствии распорядка дня. Формализация даёт онтологический якорь для этого failure mode.

---

## 2. Четыре субслоя физического слоя

Физический слой ст. 0 состоит из четырёх независимо измеряемых субслоёв. Каждый имеет baseline (порог saneness, не оптимума) и метрику просадки.

| # | Субслой | Что описывает | Baseline saneness |
|---|---------|---------------|-------------------|
| 1 | **Сон** | Длительность и качество ночного восстановления нервной системы | ≥7ч длительности + субъективное качество ≥3 из 5 |
| 2 | **Движение** | Базовая моторная активность тела — прогулки, лестницы, перемещения | ≥2 целевых прогулок/нед (>20 мин) ИЛИ ≥5000 шагов/день среднее |
| 3 | **Питание** | Регулярность приёмов пищи (не качество диеты — это downstream) | ≥3 приёмов пищи/день без длительных голодных пауз (>6ч в активные часы) |
| 4 | **Восстановление** | Субъективная энергия как интегральный сигнал работоспособности | Самооценка ≥3 из 5 на конец дня (в среднем по неделе) |

**Критерий «baseline, не оптимум»:** baseline saneness — это порог, ниже которого организм перестаёт быть надёжным носителем программы ЛР. Это не идеал сна (8.5ч + REM-фаза), не идеал движения (10000 шагов + силовая тренировка), не идеал питания (макронутриенты + микроэлементы). Это «работает достаточно для входа в программу».

**Независимость субслоев:** просадка одного субслоя не компенсируется другими. Идеальный сон не компенсирует отсутствие движения; обильное питание не компенсирует недосып.

## 3. Метрики saneness и источники данных

Метрики проектируются под доступные пилоту инструменты (не требует новой Pack-инфраструктуры).

| Субслой | Метрика | Источник данных |
|---------|---------|-----------------|
| Сон | Длительность ночного сна (часы); субъективное качество (1-5) | Pillow (приложение iOS для измерения сна), запись фактической длительности + утренняя оценка |
| Движение | Целевая прогулка (да/нет, длительность); шаги/день | Отметка в Day Close; iPhone Health (шаги) |
| Питание | Количество приёмов пищи/день; максимальная голодная пауза | Отметка в Day Close (свободная форма) |
| Восстановление | Субъективная энергия 1-5 в конце дня | Day Close (новое поле «энергия» 1-5) |

**Принцип учёта:** факты регистрируются, не интерпретируются. Метод регистрации — специализация PD.METHOD.001 (Time Accounting) для физического слоя.

## 4. Триггер реактивации программы (просадка субслоя)

Триггер срабатывает, когда **один из 4 субслоев** упал ниже baseline **2 дня подряд** в текущей неделе.

**Действие:** пауза слотов саморазвития на 1 день, восстановление просевшего субслоя в приоритет. Программа ЛР возвращается в работу после восстановления.

**Принцип «1 день пауза, не больше»:** длинная пауза превращается в дропаут (PD.FAIL.069 «Fragility Under Learning Difficulty»). Реактивация должна быть инженерной — короткой, целевой, с возвратом в ритм.

**Что значит «восстановление в приоритет»:**
- Просел сон 2 дня → следующий вечер: ранний отбой, отказ от вечерних слотов.
- Просело движение 2 дня → следующий день: целевая прогулка приоритетнее слота.
- Просело питание 2 дня → следующий день: чёткое распределение приёмов пищи приоритетнее слота.
- Просело восстановление 2 дня → следующий день: досуг приоритетнее слота (PD.METHOD.006 Leisure Organization).

## 5. Гейт перехода ст. 0 → ст. 1

**Условие:** все 4 субслоя стабильно ≥ baseline **2 недели подряд**.

**Без gate:** программа ЛР работает в downshift-режиме — микро-шаги, минимальная нагрузка, без расчёта на устойчивый ритм. Это легитимный режим (PD.METHOD.015 «First Development Slot»), но он не закрывает gate перехода Случайный → Практикующий (PD.FORM.080 ст. 1 → ст. 2).

**Проверка gate:** Диагност (MIM.R.009, см. PD.ROLE.006) сверяет 14-дневное окно данных по 4 субслоям. Если все 4 на baseline 14/14 дней — gate открыт. Если просадка в любом субслое >2 дней — gate закрыт, программа в downshift-режиме.

**Связь с двойным gate ст. 1 → ст. 2 (PD.FORM.097 §4):** ст. 0 → ст. 1 = pre-condition (saneness); ст. 1 → ст. 2 = метрический + мировоззренческий gate. Без закрытия ст. 0 → ст. 1 закрытие ст. 1 → ст. 2 нестабильно.

## 6. Граница с PACK-personal / personal-development

Физический слой ст. 0 — **снаружи** программы личного развития. Внутри программы — методы рабочего времени (PD.METHOD.001), обучения (PD.METHOD.002-005), рефлексии (PD.METHOD.028), стратегирования (PD.METHOD.008).

**Что физический слой ст. 0 не делает:**

- Не предписывает диету — это компетенция диетолога (downstream, вне Pack).
- Не предписывает тренировочный план — это компетенция тренера (downstream, вне Pack).
- Не предписывает медицинские интервенции — это компетенция врача (downstream, вне Pack).
- Не описывает методы оздоровления (йога, биохакинг, голодание) — это отдельная область знания (вне фокуса PACK-personal).

**Что физический слой ст. 0 делает:** проверяет, что носитель программы (организм) находится в saneness-режиме. Сигнализирует, когда saneness нарушен. Открывает gate перехода в программу.

**Аналогия со SoTA-границей PACK-personal:** документ-манифест ([00-pack-manifest.md](../../00-pack-manifest.md) §«Out of Scope») фиксирует, что Pack не покрывает терапию/клиническую практику. Физический слой ст. 0 — engineering-уровень saneness; clinical-уровень (терапия, лечение, спорт высоких достижений) остаётся в downstream-областях.

## 8. Антипаттерны применения

| Антипаттерн | Описание | Следствие |
|-------------|----------|-----------|
| **Физический слой как первая ступень программы** | Чтение ст. 0 как «нулевой ступени» программы ЛР | Программа ЛР начинает заниматься оздоровлением — drift домена |
| **Подмена методами оздоровления** | Использование PD.FORM.138 как обоснования для рекомендаций йоги/диеты/тренировок | Pack начинает дублировать downstream (тренер/диетолог/врач) |
| **Перфекционизм saneness** | Чтение baseline как «оптимум сна = 8.5ч + REM» | Gate ст. 0 → ст. 1 не закрывается никогда; программа не стартует |
| **Усреднение между субслоями** | «Сон плохой, но движение отличное — saneness в среднем ОК» | Скрытая просадка субслоя; ритм 10ч/нед рушится без видимой причины |
| **Длинная пауза при просадке** | Просадка субслоя → «возьму неделю отдохнуть от программы» | Дропаут вместо реактивации; FAIL.069 «Fragility Under Learning Difficulty» |

## 9. Открытые вопросы (для последующих фаз)

1. **Калибровка baseline по индивидуальной норме.** Текущие baseline универсальны (7ч сна, 2 прогулки/нед). Индивидуальная норма может отличаться (хронотип, возраст, профессия — см. PD.FORM.073 «Chronotype Synchrony Effect»). Метод калибровки baseline под пилота — следующая фаза.
2. **Интеграция Pillow через MCP.** Сейчас данные сна заходят вручную (пилот смотрит Pillow, отмечает в Day Close). Автоматическая интеграция Pillow → ЦД через MCP — техническая задача (вне Pack, в DS).
3. **Расширение метрики восстановления.** Субъективная энергия 1-5 — minimal viable proxy. Более точные proxy (HRV, ECG, sleep stages) требуют wearable-устройств и валидации — открытая фаза.
4. **Связь с PD.FORM.038 (Productive State Scale).** Шкала продуктивного состояния (0-5) пересекается с метрикой восстановления (1-5). Возможно, метрика «восстановление» = операционализация шкалы PD.FORM.038 для физического слоя. Требует исследования.
