# Принцип: Непрерывная перепрошивка (Continuous Reflash)

## 1. Формулировка

Развитие созидателя должно быть **непрерывным, инкрементальным и бесконечным** — никогда «одноразовый курс».

Инфраструктура развития (ежедневные слоты обновления, экзокортекс, недельные инкременты, ретроспективы) заменяет модель «курс как продукт» на модель «персональный release management».

## 2. Единица развития

Не «курс», а: **ежедневный слот + недельный инкремент + месячный обзор**.

## 3. Метафора

Созидатель без инфраструктуры непрерывного развития — как телефон без OTA-обновлений: работает сегодня, но деградирует относительно изменяющейся среды.

## 4. Индикаторы нарушения

- «Я прошёл курс» как завершённое событие (одноразовый релиз без поддержки)
- Нет защищённого ежедневного слота для развития
- Нет видимого недельного инкремента
- Нет цикла ретроспективы
