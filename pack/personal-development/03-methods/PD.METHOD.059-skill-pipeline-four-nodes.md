# PD.METHOD.059 — Skill Pipeline Four Nodes (Конвейер навыка: 4 узла)

## Определение

Диагностический метод поиска точки пробуксовки в процессе формирования навыка. Навык описывается как система из четырёх последовательных узлов; сбой в каждом даёт специфические симптомы и требует специфического исправления.

## Четыре узла и их поломки

| Узел | Вопрос | Поломки |
|------|--------|---------|
| **1. Замысел** | Хочу ли я это вообще? | Неверная мотивация; конфликт ценностей; чужой контекст («надо», не «хочу») |
| **2. Запуск** | Превращаю ли хотение в действие сегодня? | Нет Implementation Intention (Gollwitzer); перегрев (too much too fast); нет конкретного триггера |
| **3. Среда** | Держится ли действие без ежедневного усилия? | Нет habit-stack; reward too delayed; нет friction reduction; среда требует силы воли постоянно |
| **4. Рост** | Становится ли навык лучше, а не просто регулярнее? | Нет feedback loop; нет deliberate-practice структуры; плато без осознанного усложнения |

## IPO

- **Вход:** навык, который «не идёт» (прекратился, не прижился, топчется на месте)
- **Процесс:** последовательно задать 4 диагностических вопроса; найти первый узел с «нет»
- **Выход:** идентифицированный узел-сбой + конкретный механизм пробуксовки из 14 предусмотренных

## Ключевой инсайт

«Мы виним не тот узел» — при пробуксовке навыка первая мысль: нет дисциплины (узел «замысел»). Но чаще сбой в узле «среда» (нет habit-stack, высокий friction) или «запуск» (нет триггера). Самообвинение в отсутствии воли блокирует поиск реального узла.

## Применение

Тест: «Действие прекратилось — в каком узле сбой?»

1. Ещё хочу этот навык? → нет: работа с узлом 1 (пересмотр мотивации)
2. Хочу, но не начинаю? → нет: работа с узлом 2 (Implementation Intention, триггер)
3. Начинаю, но не удерживается регулярность? → нет: работа с узлом 3 (среда, habit-stack)
4. Регулярно, но не растёт? → нет: работа с узлом 4 (deliberate practice, feedback)
