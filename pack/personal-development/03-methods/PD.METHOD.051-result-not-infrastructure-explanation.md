# PD.METHOD.051 — «Блюдо, не кухня»: объяснение через результат, а не инфраструктуру

## Описание

При объяснении платформенных или системных концепций широкой аудитории — показывать только конечный результат (что человек получает), не механизм его производства. «Кухня» (ингредиенты, источники, слои, инфраструктура) скрыта; «блюдо» (готовый продукт взаимодействия) — на переднем плане.

## IPO

**Input:** объяснение системы/платформы + широкая (неэкспертная) аудитория

**Process:**
1. Для каждой детали объяснения задать тест: «нужна ли аудитории эта деталь, чтобы принять решение о пользе?»
2. Если нет → убрать деталь в «кухню» (не показывать)
3. Если да → оставить в «блюде» с конкретным примером
4. Проверить: текст объясняет только конечный результат взаимодействия?

**Output:** объяснение, сфокусированное на результате для потребителя, без видимой инфраструктуры

## Тест применимости

«Нужна ли аудитории эта деталь, чтобы принять решение о пользе?» Нет → убрать.

## Пример

WP-442 C-032: первоначальный текст показывал «кухню» (Pack, FPF, ингредиенты: соль, мука) и «рынок» (откуда берутся ингредиенты). После применения метода — только «блюдо»: что именно человек получает при взаимодействии с системой. Итерации: Pack/FPF → поваренная книга → кухня убрана → конкретика вместо абстракций.

## Применимость

Любое объяснение платформ, систем или методологий для аудитории, которой нужен результат, а не архитектура.
