# Метод: Управление тремя потоками созидателя

## 1. Определение

Управление тремя потоками — метод диагностики и оптимизации личной системы через линзу теории ограничений (TOC). Созидатель управляет одновременно тремя потоками: **рабочих продуктов**, **информации** и **энергии**. Производительность системы определяется узким местом — тем потоком, который ограничивает остальные.

Ключевое различение: оптимизировать можно только узкое место. Ускорение потоков, которые уже не ограничивают, не даёт прироста — система всё равно упирается в бутылочное горлышко.

Метод НЕ является:
- Тайм-менеджментом (управление временем — частный случай потока рабочих продуктов)
- Управлением энергией изолированно (энергия влияет на другие потоки)
- Методом расстановки приоритетов задач

## 2. Три потока

| Поток | Что включает | Узкое место выглядит как |
|-------|-------------|-------------------------|
| **Рабочие продукты** | Артефакты, задачи, обязательства, РП, коммиты | Задачи накапливаются, но не завершаются; «незавершёнка» растёт; решения не фиксируются |
| **Информация** | Чтение, обучение, разборы, черновики, знания | Инбокс растёт без разбора; читаю много, но ничего не применяю; знания не конвертируются в артефакты |
| **Энергия** | Состояние (FORM.038), физический ресурс, смысловой ресурс | Хронически на уровне 2-3; быстро устаю от рабочих задач; откладываю важные задачи на «когда будут силы» |

## 3. Алгоритм применения

### Шаг 1. Диагностика потоков

Задать по два вопроса на каждый поток:

**Рабочие продукты:**
- «Сколько задач и РП застряли в статусе "в процессе" более 2 недель?»
- «Завершаю ли я начатые задачи или переключаюсь на новые?»

**Информация:**
- «Растёт ли список непрочитанного / неразобранного быстрее, чем я его разбираю?»
- «Когда последний раз знание из чтения стало рабочим продуктом?»

**Энергия:**
- «Сколько дней на прошлой неделе я работал на уровне 4-5 по шкале состояния?»
- «Есть ли задачи, которые я откладываю из-за нехватки «сил», а не времени?»

### Шаг 2. Определение узкого места

Поток с наибольшим накоплением (незавершёнка / непрочитанное / дни на низком состоянии) — узкое место. Типичные профили:

| Профиль | Узкое место | Симптом |
|---------|------------|---------|
| «Много читаю, мало делаю» | Информация | Большой инбокс, мало завершённых артефактов |
| «Устаю раньше, чем заканчиваю» | Энергия | Качественная работа только утром; вечером — механическое выполнение |
| «Тону в задачах» | Рабочие продукты | Много в работе, мало закрыто; каждый день добавляется больше, чем закрывается |

### Шаг 3. Работа с узким местом

**Если узкое место — рабочие продукты:**
- Ввести лимит «незавершёнки»: не начинать новое, пока незакрытых задач больше N (обычно N=5-7)
- Завести ритуал еженедельного закрытия: какие задачи можно закрыть или отложить?
- Проверить: задачи завершаются или переходят в «вечный WIP»?

**Если узкое место — информация:**
- Ввести соотношение capture:release = 1:1 (на каждую прочитанную единицу — один артефакт)
- Использовать творческий конвейер (PD.METHOD.019): разбирать инбокс регулярно, не накапливать
- Уменьшить поток входящего: меньше подписок, более строгий фильтр качества

**Если узкое место — энергия:**
- Применить метод работы с состоянием (PD.METHOD.022): диагностировать, какой из трёх ресурсов ограничивает (физический / когнитивный / смысловой)
- Временно снизить нагрузку на другие потоки — освободить ресурс для восстановления
- Не пытаться «через силу» компенсировать энергетическое узкое место производительностью

### Шаг 4. Ревью через 2 недели

Проверить: уменьшилось ли накопление в узком потоке? Если да — искать следующее узкое место (система сбалансировалась, ограничение перешло в другой поток).

## 4. Антипаттерны / частые ошибки

| Антипаттерн | Симптом | Корень |
|-------------|---------|--------|
| Оптимизация всех потоков одновременно | «Хочу читать больше, делать больше и чувствовать себя лучше сразу» | Игнорирование логики ограничений: улучшение не-узкого места не даёт прироста |
| Ускорение узкого места через нагрузку | «Буду работать дольше» при энергетическом ограничении | Это усиливает ограничение, а не снимает его |
| Нет измерений | Диагностика на ощущениях без данных | Теория ограничений работает только при наблюдении фактического потока |
| Переключение до закрытия | Начинает работу с новым потоком, не закрыв узкое место | Новое ограничение появляется раньше, чем снято текущее |

## 5. Связь со ступенями ЛР

| Ступень | Применение метода |
|---------|-----------------|
| Случайный (1) | Потоки не различаются; метод ещё не применим |
| Практикующий (2) | Начинает различать потоки; диагностика возможна, но данных мало |
| Систематический (3) | Первое полное применение цикла диагностики; обычно обнаруживается информационный поток как узкое место |
| Дисциплинированный (4) | Все три потока измеряются; перекалибровка при смене узкого места — привычна |
| Проактивный (5) | Управляет потоками как архитектурным решением; помогает другим диагностировать их системы |

**Замечание:** метод становится полноценным инструментом только на ступени 3, когда есть достаточно данных о собственных потоках (учёт времени, инбокс, шкала состояния). На ступени 1-2 — используется как ориентировочная линза, а не как точный инструмент.
