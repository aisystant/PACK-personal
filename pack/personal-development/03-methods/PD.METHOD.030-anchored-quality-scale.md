# [PD.METHOD.030] Шкала качества с якорями на каждом делении

## Суть метода

Пятибалльная (или N-балльная) шкала самооценки, где **каждое деление имеет явный поведенческий якорь** — не только полюса (1 = плохо, 5 = отлично), но и промежуточные значения (особенно 2 и 4, определяющие положение порога).

## Зачем

При шкале с якорями только на полюсах (1, 5) человек интерпретирует средние значения **по эмоциональной интенсивности**:
- 4 = «выложился, доволен»
- 3 = «нормально»
- 2 = «не очень»

Это ломает любой расчёт, опирающийся на оценку как факт. Например: `worldview_score += 4 if q >= 3`. При якорях-полюсах оценщик ставит 3 «когда нормально» — без проверки, действительно ли его рефлексия достигла операциональной глубины (а это и есть критерий q=3).

## Алгоритм проектирования шкалы

### Шаг 1. Определить, для чего используется оценка

Шкала, влияющая на расчёт (gate, бонус, агрегат) → требует якорей на каждом делении.
Шкала исключительно для самонаблюдения → можно ограничиться полюсами.

### Шаг 2. Сформулировать якорь для каждого деления

| Деление | Что содержит якорь |
|---|---|
| **1** | Минимум — отсутствие действия / поверхностная отметка |
| **2** | Действие выполнено формально — без рефлексии глубже первого слоя |
| **3** | **Порог** — операциональная глубина достигнута (этот уровень даёт бонус/проходит gate) |
| **4** | Глубокая рефлексия + явная связь с следующим действием |
| **5** | Прорыв — переосмысление модели, влияет на план следующего цикла |

**Особое внимание:** деления **2 и 4** — они определяют, где проходит порог. От их формулировки зависит, что попадёт в «выше порога», а что — нет.

### Шаг 3. Включить оценку в физический ритуал

Чтобы шкала использовалась, она должна быть в повседневном артефакте. Варианты:
- **q:N в commit message** (как в Week Close): парсится из git-истории, не требует отдельной таблицы
- **inline в DayPlan/WeekPlan markdown** под фиксированным ключом
- **frontmatter поле** в session-summary.md

### Шаг 4. Завести аудит расхождений

Раз в N циклов (Month Close) — проверить распределение оценок:
- Все 3 → шкала не дискриминирует (якоря 2/4 нечёткие)
- Все 5 → восприятие ≠ якорю (нужно ужесточить)
- Bimodal 1/5 → пользователь не использует промежуточные

## Где работает

| Контекст | Применение |
|---|---|
| Week Close — рефлексия итогов | `q:N` в commit, влияет на worldview_score (bonus ×4 при q≥3) |
| Day Close — оценка собранности | `f:N` в commit, входит в M2 (collected days) |
| Lesson Close (программа ЛР) | `q:N` для глубины проработки занятия |
| Stage_evaluator self-input | калибровка собственной оценки vs auto-calc |

## Антипаттерны

- **«Универсальная пятибалльная шкала»:** одна шкала на все ритуалы без якорей-под-контекст
- **Якоря только на 1 и 5:** «1 = ужасно, 5 = идеально» — оценщик ставит по эмоции
- **Шкала без расчёта:** оценка собирается, но никуда не идёт — бесполезна
- **Шкала без аудита распределения:** drift якорей со временем не обнаруживается
