# Метод: Понятийная работа

## 1. Назначение

Превратить **термин** (слово с определением) в **понятие** (узел в собственном концепт-графе со связями к другим понятиям). Без этого знание остаётся в режиме «процитировал → не применяю», что является типичным failure mode Интеллектуала (PD.FAIL.070).

**Когда применяется:**
- При освоении нового материала (книга, курс, статья)
- При создании Pack-черновика собственной предметной области
- При попытке применить понятие в новой ситуации (бридж к моделированию, METHOD.038)
- При обнаружении нестыковки в собственных рассуждениях (PD.FORM.099 маркер #4)

---

## Forces

_(Optional, WP-448 Ф12) Какие конкурирующие давления удерживает метод._

| Force | Tension |
|-------|---------|
| Связывание (операция 3, ≥3 связи) ↔ Заземление (операция 4, 3 якоря) | Глубина сети понятий конкурирует с глубиной заземления в материальном: время на построение связей не потрачено на поиск якорей, и наоборот |
| Скорость охвата материала ↔ качество перехода термин→понятие | При чтении можно либо охватить много терминов поверхностно, либо довести немногие термины до уровня 3 (Навык) через все 5 операций |
| Точность цитаты источника (операция 2, извлечение определения) ↔ применение в новой ситуации (операция 5, тест) | Точное воспроизведение определения источника конкурирует с гибким применением понятия за пределами того, что источник буквально утверждал |

---

## 2. Структура работы

Понятие создаётся через **5 операций**:

1. **Идентификация термина** — выделить слово или конструкцию из текста/речи
2. **Извлечение определения** — найти, как термин формально определён в источнике (избежать «общее знание»)
3. **Связывание** — найти ≥3 связи с другими освоенными понятиями (часть/целое, частный случай/обобщение, причина/следствие, противоположность, аналог)
4. **Заземление** (см. PD.METHOD.034) — три якоря: материальный объект / действие / признак
5. **Тест применения** — применить понятие к ≥1 новой ситуации, не описанной в источнике

**Без любой операции** понятие остаётся в режиме термина и при применении в реальной задаче «не срабатывает».

---

## 3. Применение

| Момент | Что делать |
|--------|------------|
| При чтении книги | После каждой главы — 3-5 ключевых понятий пройти через 5 операций; записать связи в экзокортекс |
| При создании Pack | Каждая новая запись начинается с операций 1-3; HG-2 запрещает Method без distinction → связь обязательна |
| При разборе с наставником | Презентовать понятие через связи в своём графе, не через цитату |
| При защите квалификации (Стратег/Специалист ШСМ) | Объяснить понятие через ≥3 связи + применение в новой ситуации |

---

## 4. Уровни освоения метода (Bloom для метода)

| Уровень | Признак |
|---|---|
| 1. Объяснение | Может пересказать определение |
| 2. Умение | Применяет понятие в знакомой ситуации с подсказкой |
| 3. Навык | Связал понятие с ≥3 другими в собственном графе; применяет в новых задачах |
| 4. Мастерство | Создаёт новые понятия по необходимости; обосновывает их через FPF |

Целевой уровень для выхода из «фазы Интеллектуала» программы РР (ст. Специалист → Практик): **3 (Навык)** на ≥10 ключевых понятиях своего домена.

## Bias-Annotation (tentative)

_(Optional, WP-448 Ф12) Куда систематически съезжает внимание практикующего. Не дублирует «Failure Modes» — там «что пойдёт не так», здесь «куда дрейфует внимание»._

| Bias | Direction of distortion |
|------|--------------------------|
| Количество связей вместо их качества | Тест «≥3 связи» (операция 3) превращается в счётчик: внимание съезжает на формальные, слабые связи, лишь бы было три, не проверяя, выдерживают ли они операцию 5 (тест применения) |
| Связывание вместо теста применения | Построение сети связей ощущается как завершение работы; внимание смещается с операции 5 (тест на новой ситуации) на уже сделанную операцию 3, и построенная сеть принимается за доказательство освоения |

---

## 6. Типичные failure modes

| Failure | Описание | Лекарство |
|---|---|---|
| **Понятие = цитата** | Воспроизводит определение слово в слово, не строит связей | Тест на ≥3 связи; без связей понятие не освоено |
| **Терминологический паразит** | Использует понятие как украшение речи без применения | Тест переноса: применить к новой ситуации |
| **Связи без заземления** | Связи построены, но понятие не «живёт» в работе | Дополнить методом 034 (заземление) |
| **Понятие без теста** | Освоил в теории, не применил в реальной задаче | Операция 5 обязательна |
