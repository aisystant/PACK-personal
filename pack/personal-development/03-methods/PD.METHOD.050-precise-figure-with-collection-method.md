# PD.METHOD.050 — Точная цифра с методом сбора ≻ расплывчатая «порядок величин»

## Описание

При включении количественного утверждения в пост, отчёт или публичное заявление: если цифра точная — добавить метод сбора одной придаточной фразой. Если метод нельзя описать одной фразой — только тогда убирать цифру. Расплывчатая «порядок величин» без метода создаёт два дефекта одновременно.

## Forces

_(Optional, WP-448 Ф12) Какие конкурирующие давления удерживает метод._

| Force | Tension |
|-------|---------|
| Точность цифры ↔ гладкость расплывчатой оценки | «Около миллиона» звучит увереннее и читается легче, чем «1,29 млн (метод: X, согласие: Y)» (§«Пример»), но точная формулировка конкурирует за внимание читателя именно с этой более гладкой, но неверифицируемой альтернативой |
| Компактность формулировки (одна придаточная фраза) ↔ полнота описания метода | Process требует уместить метод сбора в одну фразу (шаг 1), но реальный метод сбора не всегда сжимается без потери точности, что конкурирует с требованием тогда убрать цифру вовсе (шаг 2) |
| Верифицируемость для читателя ↔ риск оспаривания для автора | Добавление метода сбора делает утверждение проверяемым (§«Тест применимости»), но тем самым открывает автору риск оспаривания — сама защита от непроверяемости увеличивает экспозицию ко второму давлению |

## IPO

**Input:** количественное утверждение + риск оспаривания + наличие метода сбора

**Process:**
1. Риск оспаривания высок → добавить метод сбора одной придаточной фразой («...где событием считается X, согласие дано через Y»)
2. Метод нельзя описать одной фразой → убрать точную цифру, оставить только описание метода
3. Риск низок, метод очевиден читателю → цифра без дополнений допустима

**Output:** верифицируемое утверждение (цифра + метод) или описание без цифры

## Мотив: два дефекта расплывчатости

Расплывчатое число без метода создаёт:
1. **Непроверяемость:** читатель не может воспроизвести
2. **Ложная точность:** «около миллиона» — это тоже утверждение (порядок), только неверифицируемое

## Пример

WP-459 Ф5b: «1 290 000 событий» → напарник предлагал «около миллиона». Принятое решение: «1,29 млн событий (что считается событием: X; кто дал согласие: Y)» — точная цифра + метод → верифицируемо.

## Тест применимости

«Может ли читатель с теми же данными воспроизвести эту цифру?» Нет → добавить метод или убрать цифру.

## Применимость

Посты, отчёты, публичные заявления с измеримыми утверждениями.

## Bias-Annotation

_(Optional, WP-448 Ф12) Куда систематически съезжает внимание практикующего. Не дублирует «Failure Modes» — там «что пойдёт не так», здесь «куда дрейфует внимание»._

| Bias | Direction of distortion |
|------|--------------------------|
| Выбор формулировки съезжает мимо теста применимости | Решение «точная цифра или порядок величин» принимается по ощущению уверенности, минуя явный вопрос §«Тест применимости» («Может ли читатель воспроизвести эту цифру?»), который для этого и введён |
| Ложная точность недооценивается как дефект | Из двух дефектов расплывчатости (§«Мотив») внимание смещается только на непроверяемость, а то, что «около миллиона» само по себе тоже утверждение о порядке величины, недооценивается при выборе формулировки |
