# PD.METHOD.058: Два паттерна записи в авторский каталог

## Назначение

Устранить drift при ведении каталога авторских концепций, часть которых уже имеет SoT-определение в других Pack.

## Паттерны

### Паттерн A: Указатель + авторский контекст

**Когда:** концепция **имеет** SoT-определение в другом Pack.

**Структура:**

```markdown
## {Название концепции}

**SoT:** [{ID}](relative/path/to/pack-entity)
**Авторский контекст:** как автор использует концепцию в публичных текстах/постах.
```

Тело определения **не дублируется** — только ссылка и авторский слой применения.

### Паттерн B: Полный entry

**Когда:** концепция **нигде** формально не зафиксирована.

**Структура:** полное определение с frontmatter; PACK-personal = SoT.

## Проверка принадлежности к паттерну A

```bash
grep -r "{concept-name}" ~/IWE/PACK-*/pack/*/01-domain-contract/01B-distinctions.md
```

Есть совпадение → паттерн A (указатель). Нет совпадения → паттерн B (полный entry).

## Применимость

Любой авторский каталог (интеллектуальных вкладов, инструментов, методов), где часть элементов имеет внешний SoT. Избегает копирования определений с риском drift.
