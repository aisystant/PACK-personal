# PD.METHOD.058: Два паттерна записи в авторский каталог

## Назначение

Устранить drift при ведении каталога авторских концепций, часть которых уже имеет SoT-определение в других Pack.

## Forces

_(Optional, WP-448 Ф12) Какие конкурирующие давления удерживает метод._

| Force | Tension |
|-------|---------|
| Скорость записи ↔ проверка SoT | Быстрее сразу писать полное определение (Паттерн B), чем сначала выполнить проверку grep по внешним Pack — но пропуск проверки создаёт риск дублирования и drift, который метод специально устраняет |
| Компактность каталога ↔ авторский контекст | Паттерн A убирает тело определения, оставляя только ссылку + авторский слой — экономит место и защищает от drift, но авторский контекст сам по себе может незаметно повторить часть определения, если разграничение «тело vs контекст» не удерживается |
| Единственность SoT ↔ множественность Pack | Проверка (`grep` по `~/IWE/PACK-*/`) предполагает, что SoT для концепции либо есть в одном месте, либо его нет вовсе — но при росте числа Pack граница «есть/нет совпадение» может стать неоднозначной |

## Паттерны

### Паттерн A: Указатель + авторский контекст

**Когда:** концепция **имеет** SoT-определение в другом Pack.

**Структура:**

```markdown
## {Название концепции}

**SoT:** [{ID}](relative/path/to/pack-entity)
**Авторский контекст:** как автор использует концепцию в публичных текстах/постах.
```

Тело определения **не дублируется** — только ссылка и авторский слой применения.

### Паттерн B: Полный entry

**Когда:** концепция **нигде** формально не зафиксирована.

**Структура:** полное определение с frontmatter; PACK-personal = SoT.

## Проверка принадлежности к паттерну A

```bash
grep -r "{concept-name}" ~/IWE/PACK-*/pack/*/01-domain-contract/01B-distinctions.md
```

Есть совпадение → паттерн A (указатель). Нет совпадения → паттерн B (полный entry).

## Применимость

Любой авторский каталог (интеллектуальных вкладов, инструментов, методов), где часть элементов имеет внешний SoT. Избегает копирования определений с риском drift.

## Bias-Annotation

_(Optional, WP-448 Ф12) Куда систематически съезжает внимание практикующего. Не дублирует «Failure Modes» — там «что пойдёт не так», здесь «куда дрейфует внимание»._

| Bias | Direction of distortion |
|------|--------------------------|
| Определение вместо проверки | Внимание автора при добавлении новой записи в каталог съезжает на формулирование самого определения (Паттерн B по умолчанию), а не на предварительную проверку принадлежности к паттерну A — grep-проверка воспринимается как необязательный, пропускаемый шаг |
| Авторский контекст разрастается в тело | При использовании Паттерна A внимание смещается на удобство ссылки («SoT: [ID]»), и «авторский контекст» постепенно разрастается, незаметно дублируя тело определения, которое паттерн специально запрещает копировать |
