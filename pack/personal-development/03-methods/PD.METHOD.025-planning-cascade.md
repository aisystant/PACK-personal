# [PD.METHOD.025] Каскад планирования Ученика

---

## 1. Что это

**Каскад планирования** — метод организации планирования Ученика на пяти горизонтах: от программы (1–2 года) до конкретного дня. Каждый горизонт уточняет предыдущий, не противоречит ему.

**Ключевое отличие от метода «план недели»:** каскад не заменяет нижние горизонты верхними и не регенерирует план каждый день заново. Вместо этого нижние горизонты работают как **тактическое уточнение** уже согласованной недельной гипотезы.

**Применяется:** ступени 2–5. На ступени 1 (Случайный) — только горизонты «неделя» и «день» (программный и квартальный — после первой диагностики).

---

## 2. Пять горизонтов

| Горизонт | Источник | Результат | Кадровость | Кто делает |
|----------|----------|-----------|-----------|-----------|
| **Программа** | FORM.080 стадии + RCS профиль | 5 ступеней × 4 квартала ≈ 1 год | Фиксированная (пересмотр при смене ступени) | Навигатор (R27) |
| **Квартал** | Диагностика ступени + bottleneck (FORM.089) | Фокус квартала: «собрать M1» / «пробить M4» / «усилить M2» | Раз в квартал (IRT: ≤5 вопросов Диагноста) | Навигатор + Диагност (R28) |
| **Месяц** | Мемы и методы текущего этапа, квартальный фокус | 1–3 темы месяца (одна «волна» закрепления метода) | Раз в месяц (Month Close) | Навигатор → Ученик |
| **Неделя** | Квартальный фокус + месячные темы → **гипотеза роста** | План слотов + ожидаемые ДЗ (±30% гибкость) | Воскресенье — сессия с Навигатором | Навигатор + Ученик |
| **День** | Гипотеза недели + события (календарь, состояние) | Конкретные ДЗ-пакеты: помидорки, чек-ин, разбор | Утром — короткая тактическая проверка | Ученик (автономно) |

## 3. Variant B: гипотеза роста + тактика

### Принцип

**Неделя = гипотеза роста.** Формулировка: «На этой неделе фокус — [метод/тема], ожидаю [конкретный сдвиг в слоте X RCS].»

Пример: «Неделя 19: фокус — M1 первый слот. Ожидаю выход на 3 слота/нед стабильно. Ключевое ДЗ — ms-lesson-02 + PRACTICE.005.»

**День = тактика под гипотезу.** Корректировка под контекст дня: встречи, состояние, события. Тактика не меняет гипотезу, только адаптирует исполнение.

Пример: «Сегодня встреча 15–17ч → первый слот утром 9–10ч. ДЗ: только ms-lesson-02. PRACTICE.005 переносится на завтра.»

### Что НЕ делать

- **Не регенерировать ДЗ каждый день заново** (антипаттерн: Навигатор каждое утро придумывает новый план, игнорируя недельную гипотезу). Результат — рассинхрон с месячными темами, потеря линии Spaced Repetition.
- **Не менять квартальный фокус под настроение недели.** Квартал устойчив к однонедельным спадам.
- **Не планировать без гипотезы.** «Сделать как можно больше» — не гипотеза. Нет измеримого сдвига → нечего проверять в Week Close.

## 4. Алгоритм каждого горизонта

### 4.1 Программный горизонт

**Кто:** Навигатор (R27) при первом подключении Ученика.

**Алгоритм:**
1. Диагност (R28) определяет текущую ступень (FORM.089 §6.1, ≤5 вопросов)
2. Навигатор строит маршрут: ступень → следующая ступень, ориентировочный квартал
3. Маршрут фиксируется в файле `program-map.md` репо `personal-guide`
4. Пересмотр: только при подтверждённой смене ступени (gate Д→П: W≥4 + A≥4 + stage_raw≥4)

### 4.2 Квартальный горизонт

**Кто:** Навигатор + Диагност (R28) раз в квартал (при Month Close первого месяца квартала).

**Алгоритм:**
1. Диагност: срез RCS по 7 слотам (W, M1, M2, M3, M4, IT, A) — FORM.089 §4
2. Диагност: определение bottleneck (минимальный слот, блокирующий ступень)
3. Навигатор: формулирует **квартальный фокус** (1 bottleneck слот + 1 поддерживающий)
4. Квартальный фокус фиксируется в файле `profile.md` и передаётся в месячное планирование

**Связь с RCS:**
- Bottleneck слот = фокус квартала
- Поддерживающий слот = чаще всего ближайший к bottleneck (не тратить время на уже освоенные)

### 4.3 Месячный горизонт

**Кто:** Навигатор при Month Close (первый понедельник месяца).

**Алгоритм:**
1. Берём квартальный фокус
2. Определяем 1–3 темы на месяц: мемы, методы, практики из каталогов (CAT.001, CAT.003)
3. Темы расставляются в порядке curriculum: более фундаментальные — раньше
4. Фиксируются в файле `monthly/YYYY-MM.md` репо `personal-guide`

**Spaced Repetition принцип:** тема возвращается через 3–4 недели с новым контекстом. Не нужно «пройти» тему за месяц — достаточно одной «волны» касания.

### 4.4 Недельный горизонт

**Кто:** Навигатор + Ученик (воскресенье, ≤20 мин).

**Алгоритм:**
1. Навигатор читает месячные темы + итоги прошлой недели (Week Close)
2. Формулируется **гипотеза роста недели** (что ожидаем + почему именно сейчас)
3. Составляется план слотов (сколько помидорок по слотам M1/M2/M4/W)
4. Указываются конкретные ДЗ-пакеты (±30% гибкость — оставляем место под неожиданное)
5. Фиксируется в `weekly/YYYY-WNN.md` репо `personal-guide`

**Формат гипотезы роста:**
```
Фокус: [слот или тема]
Гипотеза: [конкретный сдвиг, который ожидаем]
Ключевые ДЗ: [3–5 конкретных задания с оценкой времени]
Проверка: [как узнаем, что гипотеза подтвердилась — в Week Close]
```

### 4.5 Дневной горизонт

**Кто:** Ученик (автономно, утром ≤5 мин).

**Алгоритм:**
1. Открываем `weekly/YYYY-WNN.md` — видим гипотезу и ДЗ-пакеты
2. Смотрим на сегодняшний день: встречи, состояние (FORM.038 §4.1)
3. Выбираем конкретные ДЗ из пакетов под контекст дня
4. Расставляем слоты (golden-hour первым при состоянии ≥3)
5. Записываем в `daily/YYYY-MM-DD.md` или в дневник (1–3 строки)

**Тактическая корректировка ≠ смена плана.** Если день срывается — фиксируем причину, гипотеза недели не пересматривается. Пересмотр гипотезы — только в следующей воскресной сессии.

## 5. Связь с ступенями ЛР

| Ступень | Активные горизонты | Что упрощается |
|---------|-------------------|----------------|
| 1 Случайный | Неделя + День | Программа и квартал — после первой диагностики; Навигатор ведёт за руку |
| 2 Практикующий | Неделя + День (самостоятельно) + Месяц | Впервые самостоятельная дневная тактика; Навигатор — раз в неделю |
| 3 Систематический | Все 5 горизонтов | Квартал + Программа активны; Навигатор — раз в месяц |
| 4 Дисциплинированный | Все 5 горизонтов + саморефлексия | Навигатор — раз в квартал; Ученик сам формулирует гипотезу |
| 5 Проактивный | Все 5 горизонтов + передача другим | Ученик может вести каскад для сопроизводителей |

**Принцип постепенного делегирования:** на ст. 1–2 Навигатор ведёт. На ст. 3–4 Ученик берёт автономию снизу вверх. На ст. 5 — сам Навигатор для других.

## 6. Антипаттерны

| Антипаттерн | Симптом | Правильно |
|-------------|---------|-----------|
| **Ежедневная регенерация** | Каждое утро — новый план без связи с гипотезой недели | Дневная тактика = исполнение недельного плана, не новый план |
| **Гипотеза без измерения** | «На этой неделе учусь» | Конкретный сдвиг в слоте: «M1: с 2 слотов до 4» |
| **Квартал без bottleneck** | «Квартал — общее развитие» | Один bottleneck слот + один поддерживающий |
| **Месяц без тем** | «Продолжаем в том же духе» | 1–3 конкретные темы из каталога, curriculum-упорядоченные |
| **Программа без ступени** | Нет диагностики | Диагност → ступень → маршрут → программный горизонт |
| **Частая переоценка ступени** | «Может, я уже на ступени 3?» каждые 2 недели | IRT: раз в квартал, ≤5 вопросов, по данным артефактов |
