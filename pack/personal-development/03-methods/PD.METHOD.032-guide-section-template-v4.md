# Метод: Шаблон подраздела руководства v4

## 1. Назначение

14-элементная структура для создания единообразных подразделов персонального руководства. Обеспечивает: единство восприятия, автоматическую генерацию/верификацию, связь с cp/bh-индикаторами.

## Forces

_(Optional, WP-448 Ф12) Какие конкурирующие давления удерживает метод._

| Force | Tension |
|-------|---------|
| Единообразие структуры (14 элементов) ↔ применимость к разным типам понятий | Шаблон фиксирует все 14 элементов ради «единства восприятия» (Назначение), но элемент #4 «Мем, который снимается» предполагает не каждое понятие имеет блокирующий мем (см. PD.METHOD.033, факультативно для технических понятий) — единообразие тянет заполнять все 14 пунктов даже там, где содержательно нечего вписывать |
| Frontmatter-якоря (машиночитаемость) ↔ 14-элементный контент (прозаическая полнота) | `cp_check`/`bh_check` привязывают подраздел к одному слоту диагностики, но содержание («Степени мастерства» #10, «На практике» #12) может затрагивать несколько срезов сразу — компактность машинного тега конкурирует с многомерностью содержания |
| Фиксированный порядок элементов ↔ индивидуальная траектория читателя | Порядок #4→#5 (мем-снятие перед определением) закреплён как правило для всех подразделов, но элемент #14 «Что дальше» предполагает нелинейную траекторию между подразделами — жёсткая внутренняя последовательность соседствует с изменчивой внешней |

## 2. Frontmatter-якоря подраздела

Каждый подраздел содержит в frontmatter:
- **`subsection_id`**: уникальный ID вида `S{N}.{NN}` (номер секции + порядковый)
- **`cp_check`**: какой cp-слот активирует (cp.rhy, cp.wld, cp.skl, cp.iwe, cp.int, cp.agt)
- **`bh_check`**: какой behavioral-индикатор связан (bh.sys, bh.met, bh.inv, bh.awr)
- **`bottleneck_hint`**: встроенная подсказка для застрявшего (текст прямо в уроке, без обращения к наставнику)
- **`stage_relevant`**: для каких ступеней (1-5) раздел наиболее актуален

## 3. 14-элементная структура контента

| № | Элемент | Назначение |
|---|---------|-----------|
| 1 | **Время** | Ориентировочное время освоения |
| 2 | **В одном предложении** | Ключевая идея раздела |
| 3 | **Сигнатура понятий** | Термины и их взаимосвязь |
| 4 | **Мем, который снимается** | Блокирующее убеждение — снять до определения |
| 5 | **Определение из источника** | Каноническое определение с источником |
| 6 | **Развитие мысли** | Контекст, нюансы, следствия |
| 7 | **Метод — минимальный шаг** | Конкретное действие (≤30 мин) |
| 8 | **Пример из жизни** | Иллюстрация из реальной практики |
| 9 | **Типичная ошибка** | Частый failure mode |
| 10 | **Степени мастерства** | Чем отличается ступень 1 от 3 от 5 |
| 11 | **Проверка себя** | Чеклист самодиагностики |
| 12 | **На практике** | Как встроить в рабочий ритм |
| 13 | **См. также** | Связанные подразделы и методы |
| 14 | **Что дальше** | Следующий подраздел в траектории |

## 4. Применение

- Использовать при создании/обновлении подразделов персонального руководства (personal-guide)
- 37 подразделов (s6-s10) созданы по этому шаблону (commit 5bd37f5, май 2026)
- Порядок элементов фиксирован: мем-снятие (#4) предшествует определению (#5)
- `bottleneck_hint` (#3 frontmatter) = минимум для самостоятельного выхода без наставника

## Bias-Annotation

_(Optional, WP-448 Ф12) Куда систематически съезжает внимание практикующего. Не дублирует «Failure Modes» — там «что пойдёт не так», здесь «куда дрейфует внимание»._

| Bias | Direction of distortion |
|------|--------------------------|
| Порядок элементов воспринимается как чеклист, не как приём | Внимание съезжает с того, ЗАЧЕМ мем-снятие (#4) должно идти перед определением (#5), к формальному соблюдению нумерации — элементы заполняются по порядку номеров, а не с проверкой, снят ли блокирующий мем перед введением термина |
| Тег `cp_check` выбирается по похожести, не по проверке | При заполнении frontmatter внимание уходит на выбор одного из шести готовых значений (cp.rhy/cp.wld/cp.skl/cp.iwe/cp.int/cp.agt) как самого похожего на тему подраздела, а не на вопрос — действительно ли содержание продвигает именно этот срез, а не соседний |
