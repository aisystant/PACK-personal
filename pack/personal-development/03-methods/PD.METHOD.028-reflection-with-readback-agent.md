# [PD.METHOD.028] Рефлексия с read-back агентом

## Суть метода

Пилот заполняет структурированную форму рефлексии (5 вопросов, 5 минут). Агент Портной читает ответы при следующем рендере персонального руководства и учитывает их в двух точках:

- **Ответ 3** → проверка gate перехода фаз (PD.FORM.087)
- **Ответ 5** → входной контекст для tomorrow's daily

## Механика

1. При первом рендере (Шаг 6.8) `reflection-template.md` копируется в `history/<дата>-reflection.md`
2. Пилот заполняет 5 вопросов после занятия или дня
3. При следующем render Портной читает `history/*-reflection.md` за 7 дней
4. Портной применяет ответы к двум точкам маршрутизации (ответ 3 и ответ 5)

## Ключевой паттерн

**«Мини-форма → read-back агентом»** = асинхронная обратная связь без синхронного диалога. Применимо к любому циклическому процессу (ревью, daily close), где нужен структурированный input от пользователя с последующим учётом агентом.

## Инварианты

- Форма заполняется пилотом, не агентом
- Read-back — в отдельном цикле (не в момент заполнения)
- Архив history/ накапливается, не перезаписывается

## Применимость

Не специфичен для personal-guide. Применим к week-close ревью, month-close самооценке, любому персональному логу с необходимостью агентного учёта.
