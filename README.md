# SPF Personal

**Репозиторий содержит SPF (Second Principles Framework) и один доменный Pack — «Характеристики и состояния созидателя»**

---

## Зачем это нужно

В эпоху ИИ формализация знаний стала критичной. ИИ может существенно ускорить понимание области **при наличии структуры знания**, а при отсутствии формализованной структуры ИИ закрепляет путаницу.

Этот репозиторий — инструмент для тех, кто хочет:
- сохранять профессиональную востребованность
- быстрее обучать и масштабировать
- строить ИИ-помощников, которые «понимают вашу предметность»

**Подробнее**: [docs/fpf-spf-pack.md](docs/fpf-spf-pack.md) — полная концептуальная модель

---

## Что здесь находится

Этот репозиторий содержит **два слоя**:

### 1. SPF (Second Principles Framework) — фреймворк вторых принципов

SPF задаёт **как оформлять вторые принципы** предметной области так, чтобы получалось инженерное знание:
- форму (структуру pack'а)
- процесс (производство знаний)
- guardrails (lint, гейты качества)

**SPF основан на FPF** (First Principles Framework) — фреймворке первых принципов.

### 2. Pack «Характеристики и состояния созидателя» — паспорт предметной области

**Pack** (он же **паспорт области**) — формализованное знание о конкретном домене.

| Pack — это | Pack — это НЕ |
|------------|---------------|
| Source-of-truth (инженерное ядро знания) | Курс или учебник |
| Вторые принципы, доведённые до инженерного состояния | Маршрут развития |
| Артефакт знания | Набор публикаций |

---

## Иерархия уровней

```
FPF (Level 1)  →  фреймворк первых принципов (мета-онтология + различения)
       ↓
SPF (Level 2)  →  фреймворк вторых принципов (форма + процесс)
       ↓
Pack           →  формализованное знание конкретной области (паспорт области)
       ↓
Downstream     →  производные представления (курсы, AI-агенты, руководства)
```

| Уровень | Что | Универсальность |
|---------|-----|-----------------|
| **FPF** | Мета-онтология + язык различений | Универсален по содержанию |
| **SPF** | Форма + процесс | Универсален по форме и процессу |
| **Pack** | Формализованное знание | Специфичен для домена |
| **Downstream** | Производные представления | Свободная форма |

## Ключевые понятия

| Термин | Определение |
|--------|-------------|
| **Первые принципы** | Универсальные различения корректного мышления о знаниях (метод ≠ инструмент, знание ≠ обучение) |
| **Вторые принципы** | Устойчивые предметные закономерности конкретной области |
| **Pack / паспорт области** | Вторые принципы, доведённые до инженерного состояния |
| **Bounded context** | Зафиксированные границы и словарь предметной области |
| **Distinction** | Различение, предотвращающее онтологическую путаницу |
| **Failure mode** | Типовая ошибка интерпретации |
| **SoTA** | Статус актуальности утверждения + критерий пересмотра |

## Предметная область: «Характеристики и состояния созидателя»

**Инженерная аналогия**: как в автопроме у изделия есть характеристики (безопасность, надёжность) и методы испытаний, так и у «созидателя» есть характеристики и методы оценки/проверки.

### Роли созидателя

| Роль | Описание |
|------|----------|
| Ученик | Режим освоения нового |
| Интеллектуал | Режим анализа и рефлексии |
| Профессионал | Режим продуктивной деятельности |
| Исследователь | Режим поиска нового знания |
| Просветитель | Режим передачи знания |

### Состояния ученика

| Состояние | Описание |
|-----------|----------|
| Случайный | Нет регулярности |
| Практикующий | Появляется регулярность |
| Систематический | Есть система, следует ей |
| Дисциплинированный | Устойчивое следование системе |
| Проактивный | Опережающие действия, инициатива |

### Характеристики (каталог)

| Характеристика | Категория | Приоритет |
|----------------|-----------|-----------|
| Агентность | Core | High |
| Калибр личности | Core | High |
| Стрессоустойчивость | Resilience | Medium |

## Связь с digital-twin-mcp

**digital-twin-mcp** выступает как внешний «измерительный контур»:
- индикаторы и их типы
- расчёт производных оценок (stage/mastery/agency/risk)
- генеративные тексты/интерпретации **не являются source-of-truth**

В pack это оформляется как **контракт интерфейса**: какие индикаторы используем и как связываем их с характеристиками/состояниями.

## Типовые ошибки (анти-паттерны)

| Ошибка | Как избежать |
|--------|--------------|
| Pack превращается в обучение | Дидактика — downstream |
| Состояния становятся «этапами» | Состояния — классы, не последовательность |
| Инструменты объявляются методами | Метод ≠ инструмент |
| SoTA превращается в «обзор статей» | SoTA = статус + revision criterion |
| Downstream становится source-of-truth | Pack — единственный source-of-truth |

## Финальная формула

> **Первые принципы** — универсальные различения корректного мышления о знаниях; они существуют независимо от формализации.

> **FPF** — фреймворк, который эксплицирует первые принципы и задаёт язык корректности.

> **Вторые принципы** — устойчивые предметные закономерности конкретной области; они существуют до формализации в профессиональном опыте.

> **SPF** — фреймворк, который задаёт, как вторые принципы могут быть зафиксированы как инженерное знание.

> **Pack / паспорт области** — вторые принципы, доведённые до инженерного состояния; это артефакт знания (source of truth), а не процесс и не обучение.
